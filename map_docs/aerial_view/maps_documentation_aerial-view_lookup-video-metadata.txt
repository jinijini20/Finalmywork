동영상의 메타데이터 검색
유효한 주소 또는 videoID가 주어지면 lookupVideoMetadata [https://developers.google.com/maps/documentation/aerial-view/reference/rest/v1/videos/lookupVideoMetadata?hl=ko] 메서드는 지정된 동영상과 연결된 메타데이터를 반환합니다. 응답 본문의 메타데이터를 사용하면 이미지의 최신성, 동영상 길이를 확인할 수 있으며 동영상의 존재를 확인하는 무료 워크플로도 제공됩니다.
API 사용 방법
URL 인코딩 [https://developers.google.com/maps/url-encoding?hl=ko]된 미국 우편 주소 또는 videoId [https://developers.google.com/maps/documentation/aerial-view/reference/rest/v1/VideoMetadata?hl=ko]를 전달하여 이 엔드포인트를 호출합니다.
curl -X GET "https://aerialview.googleapis.com/v1/videos:lookupVideoMetadata?key=YOUR_API_KEY&address=POSTAL_ADDRESS"
그러면 API는 다음 두 응답 중 하나를 반환합니다.
동영상이 있으면 lookupVideoMetadata는 동영상의 메타데이터를 반환합니다.
동영상이 없으면 API는 404 오류 코드를 반환합니다.
요청 예시
다음 코드 샘플은 600 Montgomery St, San Francisco, CA 94111에 위치한 Transamerica Pyramid의 공중 촬영 동영상을 요청합니다.
curl -X GET "https://aerialview.googleapis.com/v1/videos:lookupVideoMetadata?key=YOUR_API_KEY&address=600%20Montgomery%20St%2C%20San%20Francisco%2C%20CA%2094111"

{

}
동영상을 찾을 수 없습니다.
동영상이 Google 데이터베이스에 없습니다. 아직 렌더링되지 않았거나 위치가 지원되지 않는 지역에 있기 때문일 수 있습니다.
위치가 지원되는 지역 [https://developers.google.com/maps/documentation/aerial-view/overview?hl=ko#coverage_areas]에 있는 경우 renderVideo [https://developers.google.com/maps/documentation/aerial-view/reference/rest/v1/videos/renderVideo?hl=ko]를 사용하여 렌더링 요청을 시작할 수 있습니다.
{
  "error": {
    "code": 404,
    "message": "Video not found.",
    "status": "NOT_FOUND"
  }
}
