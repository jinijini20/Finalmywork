iOS용 Navigation SDK
bookmark_border
iOS 앱을 위한 동적 탐색 환경을 빌드합니다.
시작하기
iOS용 Navigation SDK의 일반적인 기능을 사용하여 빌드를 시작합니다.
explore [https://developers.google.com/maps/documentation/navigation/ios-sdk/config?hl=ko]
프로젝트 설정 [https://developers.google.com/maps/documentation/navigation/ios-sdk/config?hl=ko]
계정을 만들고, API 키를 생성하고, 지도 작성을 시작합니다.
map [https://developers.google.com/maps/documentation/navigation/ios-sdk/route?hl=ko]
첫 번째 경로 탐색 [https://developers.google.com/maps/documentation/navigation/ios-sdk/route?hl=ko]
Navigation SDK를 사용하여 앱 내에서 단일 대상 경로를 탐색하는 방법을 알아봅니다.
palette [https://developers.google.com/maps/documentation/navigation/ios-sdk/controls?hl=ko]
탐색 UI 수정 [https://developers.google.com/maps/documentation/navigation/ios-sdk/controls?hl=ko]
앱의 탐색 환경 중에 어떤 UI 요소와 컨트롤을 표시할지 맞춤설정합니다.
chat [https://developers.google.com/maps/documentation/navigation/ios-sdk/events?hl=ko]
탐색 이벤트 리슨 [https://developers.google.com/maps/documentation/navigation/ios-sdk/events?hl=ko]
사용자가 앱을 사용하여 경로를 따라 이동할 때 발생하는 내비게이션 이벤트를 수신 대기하고 이에 응답합니다.
기능
iOS용 Navigation SDK의 핵심 기능을 알아봅니다.
Google 내비게이션 환경 [https://developers.google.com/maps/documentation/navigation/ios-sdk/intro-google-nav?hl=ko]
앱에 Google 내비게이션 환경을 추가합니다.
맞춤 탐색 환경 [https://developers.google.com/maps/documentation/navigation/ios-sdk/intro-custom-nav?hl=ko]
Google 탐색 라우팅을 활용하여 앱의 맞춤 환경을 빌드합니다.
경로 경험 [https://developers.google.com/maps/documentation/navigation/ios-sdk/intro-route-experience?hl=ko]
경로 환경설정을 맞춤설정하고, 경유지를 관리하고, 경로를 계획할 수 있습니다.

code [https://developers.google.com/maps/documentation/navigation/ios-sdk/try-demo?hl=ko]
