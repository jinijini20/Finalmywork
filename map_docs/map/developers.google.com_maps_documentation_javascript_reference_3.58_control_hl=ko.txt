StreetViewControlOptions [https://developers.google.com/maps/documentation/javascript/reference/3.58/control?hl=ko#StreetViewControlOptions] 인터페이스

| --- |
| position optional | 유형:  ControlPosition optional 위치 ID입니다. 지도에서 컨트롤의 위치를 지정하는 데 사용됩니다. 기본 위치는 탐색(확대/축소 및 화면 이동) 컨트롤 내에 삽입됩니다. 이 위치가 비어 있거나 zoomControlOptions 또는 panControlOptions에 지정된 위치와 동일하면 스트리트 뷰 컨트롤이 내비게이션 컨트롤의 일부로 표시됩니다. 그렇지 않으면 별도로 표시됩니다. |
| sources optional | 유형:  Iterable<StreetViewSource> optional 기본값: [StreetViewSource.DEFAULT] 검색할 파노라마의 소스를 지정합니다. 이렇게 하면 공식 Google 파노라마만 검색하도록 제한할 수 있습니다. 소스를 여러 개 설정하면 이러한 소스의 교집합으로 평가됩니다. 참고: StreetViewSource.OUTDOOR 소스는 현재 지원되지 않습니다. |

ControlPosition [https://developers.google.com/maps/documentation/javascript/reference/3.58/control?hl=ko#ControlPosition] 상수

google.maps.ControlPosition 상수

지도에서 컨트롤의 게재위치를 지정하는 데 사용되는 식별자입니다.  컨트롤은 같은 레이아웃 위치에 있는 다른 컨트롤을 기준으로 배치됩니다.  먼저 추가된 컨트롤은 지도의 모서리 가까이에 배치됩니다.  왼쪽에서 오른쪽(LTR) 레이아웃 컨텍스트와 오른쪽에서 왼쪽 (RTL) 레이아웃 컨텍스트를 모두 자동으로 지원하려면 '논리적 값'(https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_logical_properties_and_values [https://developer.mozilla.org/en-US/docs/Web/CSS/CSS_logical_properties_and_values] 참고)을 사용하는 것이 좋습니다. LTR의 논리 값:  +----------------+ | BSIS BSIC BSIE | | ISBS      IEBS | |                | | ISBC      IEBC | |                | | ISBE      IEBE | | BEIS BEIC BEIE | +----------------+ RTL의 논리 값: +----------------+ | BSIE BSIC BSIS | | IEBS      ISBS | |                | | IEBC      ISBC | |                | | IEBE      ISBE | | BEIE BEIC BEIS | +----------------+ 기존 값: +----------------+ | TL    TC    TR | | LT          RT | |                | | LC          RC | |                | | LB          RB | | BL    BC    BR | +----------------+ 상단 또는 하단 행의 요소가 행의 가운데를 향해 흐릅니다. 왼쪽 또는 오른쪽 열의 요소가 열 중앙을 향해 흐릅니다.
const {ControlPosition} = await google.maps.importLibrary("core")를 호출하여 액세스합니다. Maps JavaScript API의 라이브러리 [https://developers.google.com/maps/documentation/javascript/libraries?hl=ko]를 참고하세요.
| 상수 |
| --- |
| BLOCK_END_INLINE_CENTER | LTR와 RTL 모두에서 BOTTOM_CENTER와 같습니다. |
| BLOCK_END_INLINE_END | LTR의 BOTTOM_RIGHT 또는 RTL의 BOTTOM_LEFT와 같습니다. |
| BLOCK_END_INLINE_START | LTR의 BOTTOM_LEFT 또는 RTL의 BOTTOM_RIGHT와 같습니다. |
| BLOCK_START_INLINE_CENTER | LTR과 RTL 모두에서 TOP_CENTER와 같습니다. |
| BLOCK_START_INLINE_END | LTR의 TOP_RIGHT 또는 RTL의 TOP_LEFT와 같습니다. |
| BLOCK_START_INLINE_START | LTR의 TOP_LEFT 또는 RTL의 TOP_RIGHT와 같습니다. |
| BOTTOM_CENTER | 요소는 맨 아래 행의 중심에 배치됩니다. 대신 BLOCK_END_INLINE_CENTER를 사용하는 것이 좋습니다. |
| BOTTOM_LEFT | 요소는 왼쪽 하단에 배치되고 중심을 향해 흐릅니다. 요소는 Google 로고의 오른쪽에 배치됩니다. 대신 BLOCK_END_INLINE_START를 사용하세요. |
| BOTTOM_RIGHT | 요소는 오른쪽 하단에 배치되고 중심을 향해 흐릅니다. 요소는 저작권 왼쪽에 배치됩니다. 대신 BLOCK_END_INLINE_END를 사용하세요. |
| INLINE_END_BLOCK_CENTER | LTR의 RIGHT_CENTER 또는 RTL의 LEFT_CENTER와 같습니다. |
| INLINE_END_BLOCK_END | LTR의 RIGHT_BOTTOM 또는 RTL의 LEFT_BOTTOM과 같습니다. |
| INLINE_END_BLOCK_START | LTR의 RIGHT_TOP 또는 RTL의 LEFT_TOP과 같습니다. |
| INLINE_START_BLOCK_CENTER | LTR의 LEFT_CENTER 또는 RTL의 RIGHT_CENTER와 같습니다. |
| INLINE_START_BLOCK_END | LTR의 LEFT_BOTTOM 또는 RTL의 RIGHT_BOTTOM과 같습니다. |
| INLINE_START_BLOCK_START | LTR의 LEFT_TOP 또는 RTL의 RIGHT_TOP과 같습니다. |
| LEFT_BOTTOM | 요소는 왼쪽에, 왼쪽 맨 아래 요소 위에 배치되고 위쪽으로 흐릅니다. 대신 INLINE_START_BLOCK_END를 사용하는 것이 좋습니다. |
| LEFT_CENTER | 요소는 왼쪽의 중심에 배치됩니다. 대신 INLINE_START_BLOCK_CENTER를 사용하는 것이 좋습니다. |
| LEFT_TOP | 요소는 왼쪽에, 왼쪽 맨 위 요소 아래에 배치되고 아래쪽으로 흐릅니다. 대신 INLINE_START_BLOCK_START를 사용하는 것이 좋습니다. |
| RIGHT_BOTTOM | 요소는 오른쪽에, 오른쪽 맨 아래 요소 위에 배치되고 위쪽으로 흐릅니다. 대신 INLINE_END_BLOCK_END를 사용하세요. |
| RIGHT_CENTER | 요소는 오른쪽의 중심에 배치됩니다. 대신 INLINE_END_BLOCK_CENTER를 사용하는 것이 좋습니다. |
| RIGHT_TOP | 요소는 오른쪽에, 오른쪽 맨 위 요소 아래에 배치되고 아래쪽으로 흐릅니다. 대신 INLINE_END_BLOCK_START를 사용하는 것이 좋습니다. |
| TOP_CENTER | 요소는 맨 위 행의 중심에 배치됩니다. 대신 BLOCK_START_INLINE_CENTER를 사용하는 것이 좋습니다. |
| TOP_LEFT | 요소는 왼쪽 상단에 배치되고 중심을 향해 흐릅니다. 대신 BLOCK_START_INLINE_START를 사용하는 것이 좋습니다. |
| TOP_RIGHT | 요소는 오른쪽 상단에 배치되고 중심을 향해 흐릅니다. 대신 BLOCK_START_INLINE_END를 사용하는 것이 좋습니다. |
