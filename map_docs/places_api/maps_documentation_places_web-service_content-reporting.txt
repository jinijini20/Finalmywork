부적절한 콘텐츠 신고
Places API (신규)는 응답에서 장소에 대한 정보를 반환합니다. 주소, 전화번호와 같은 사실적 콘텐츠와 함께 리뷰, 사진 등 사용자가 Google 지도에 제공한 콘텐츠도 대답에 포함될 수 있습니다. 또한 Places API (신규)용 Gemini 모델 기능을 사용하면 LLM에서 집계되고 요약된 생성형 데이터가 응답에 포함될 수 있습니다.
Google에서는 Places API (New)에서 문제가 있는 콘텐츠가 반환되기 전에 이를 식별하기 위한 일련의 조치를 구현했지만, Google에서 삭제할 수 있도록 사용자가 부적절한 콘텐츠를 신고할 수 있는 방법도 제공합니다. 이번 출시를 통해 Places API (New)에는 Places API (New) 응답에 포함된 모든 리뷰, 사진, 생성형 요약에 새로운 flagContentUri 필드가 포함됩니다. 이 필드에는 사용자가 Google에 부적절한 콘텐츠를 알릴 수 있는 고유 콘텐츠 식별자를 포함한 링크가 포함되어 있습니다.
예를 들어 장소에 대한 개별 리뷰 5개를 반환하는 요청을 합니다. 각 개별 리뷰에는 사용자가 일부 또는 모든 리뷰에 대해 문제를 신고할 수 있도록 해당 리뷰의 고유 식별자가 포함된 새로운 flagContentUri 필드가 포함됩니다.
API 탐색기를 사용하면 라이브 요청을 통해 다음 새로운 옵션을 익힐 수 있습니다.

링크 형식
flagContentUri 필드에 포함된 URL에는 콘텐츠의 고유 식별자가 포함됩니다. 이 링크로 이동하면 다음과 같은 형식의 Google 호스팅 웹페이지가 열립니다.
이 대화상자에서 사용자는 적절한 문제 유형에 대해 Google에 신고를 제출할 수 있습니다. 예를 들어 콘텐츠의 욕설, 괴롭힘 또는 법적 문제를 신고할 수 있습니다.
콘텐츠 신고 링크 표시
flagContentUri 필드는 이미 Places API (New) 응답에 포함된 다음 객체에서 사용할 수 있습니다.
Review
Photo
GenerativeSummary
NeighborhoodSummary
EvChargeAmenitySummary
ReviewSummary
ContextualContent
고객이 콘텐츠 문제를 신고할 수 있도록 이러한 객체를 표시할 때 콘텐츠 신고 링크를 표시해야 합니다.
대답에서 콘텐츠 신고 링크에 액세스
참고: GenerativeSummary, NeighborhoodSummary, EvChargeAmenitySummary, ReviewSummary를 사용하는 예는 Places API (신규)의 Gemini 모델 기능을 참고하세요.
다음 장소 세부정보 (신규) 요청에는 응답에 장소 리뷰가 포함됩니다.
curl -X GET -H 'Content-Type: application/json' \
-H "X-Goog-Api-Key: API_KEY" \
-H "X-Goog-FieldMask: id,displayName,reviews" \
https://places.googleapis.com/v1/places/ChIJj61dQgK6j4AR4GeTYWZsKWw

링크 만료
콘텐츠 링크는 지원 콘텐츠가 삭제될 때까지 활성 상태로 유지됩니다. 링크를 클릭하면 삭제된 콘텐츠의 대화상자가 계속 열리지만 이미 삭제된 콘텐츠에 대한 신고를 제출해도 아무런 효과가 없습니다.
사용해 보기
API 탐색기를 사용하면 샘플 요청을 만들어 API와 API 옵션을 익힐 수 있습니다.
페이지 오른쪽의 API 아이콘 을 선택합니다.
선택적으로 표준 매개변수 표시를 펼치고 fields 매개변수를 필드 마스크로 설정합니다.
원하는 경우 요청 본문을 수정합니다.
실행 버튼을 선택합니다. 대화상자에서 요청을 보낼 때 사용할 계정을 선택합니다.
API 탐색기 패널에서 펼치기 아이콘 을 선택하여 API 탐색기 창을 펼칩니다.
