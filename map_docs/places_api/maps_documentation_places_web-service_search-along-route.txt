경로를 따라 검색
텍스트 검색 (신규)을 사용하여 경로를 따라 검색하려면 요청에서 Routes API의 미리 계산된 경로를 폴리라인으로 전달합니다. 응답에는 검색 기준과 일치하고 지정된 경로 근처에 있는 장소가 포함됩니다.
경로를 따라 검색하려면 다음 단계를 따르세요.
Routes API를 사용하여 응답에서 경로의 인코딩된 폴리라인을 반환하는 경로를 계산합니다.
참고: 인코딩된 폴리라인 알고리즘 형식에 설명된 대로 자체 폴리라인을 인코딩할 수도 있습니다.
searchAlongRouteParameters.polyline.encodedPolyline 파라미터를 사용하여 경로의 인코딩된 폴리라인을 텍스트 검색 (신규)에 전달하여 검색 결과를 경로에 바이어스합니다. 그러면 응답에 검색 기준과 일치하고 지정된 경로 근처에 있는 장소가 포함됩니다.
API 탐색기를 사용하면 API와 API 옵션을 숙지할 수 있도록 라이브 요청을 할 수 있습니다.

다음 예에서는 경로의 인코딩된 폴리라인을 사용하여 검색 결과를 편향시킵니다.
  curl -X POST -d '{
    "textQuery" : "Spicy Vegetarian Food",
    "searchAlongRouteParameters": {
      "polyline": {
        "encodedPolyline": "ROUTE_POLYLINE"
      }
    }
  }' \
  -H 'Content-Type: application/json' -H 'X-Goog-Api-Key: API_KEY' \
  -H 'X-Goog-FieldMask: places.displayName,places.formattedAddress,places.priceLevel' \
  'https://places.googleapis.com/v1/places:searchText'

전체 경로에 대해 결과를 편향시킬 필요는 없습니다. 예를 들어 검색을 수행하기 전에 이미 경로의 중간 지점에 도달했을 수 있습니다. 이 경우 검색에 명시적 라우팅 원본을 지정하여 인코딩된 다중선에 정의된 원본을 재정의할 수 있습니다. 다음 예에서는 경로 폴리라인의 중간 지점인 캘리포니아주 샌마테오의 위도 및 경도 좌표를 지정합니다.
  curl -X POST -d '{
    "textQuery" : "Spicy Vegetarian Food",
    "searchAlongRouteParameters": {
      "polyline": {
        "encodedPolyline": "ROUTE_POLYLINE"
      }
    },
    "routingParameters": {
      "origin": {
        "latitude": 37.56617,
        "longitude": -122.30870
      }
    }
  }' \
  -H 'Content-Type: application/json' -H 'X-Goog-Api-Key: API_KEY' \
  -H 'X-Goog-FieldMask: places.displayName,places.formattedAddress,places.priceLevel' \
  'https://places.googleapis.com/v1/places:searchText'
참고: 경로를 따라 검색하면 출발지에서 목적지까지의 우회 시간이 최소인 장소가 반환되므로 경로의 출발지와 목적지가 동일하거나 서로 가까운 경우 경로를 따라 검색해도 결과가 반환되지 않을 수 있습니다.
사용해 보기
API 탐색기를 사용하면 API 및 API 옵션에 익숙해질 수 있도록 샘플 요청을 할 수 있습니다.
페이지 오른쪽에서 API 아이콘 api을 선택합니다.
원하는 경우 요청 매개변수를 수정합니다.
실행 버튼을 선택합니다. 대화상자에서 요청을 보낼 때 사용할 계정을 선택합니다.
API 탐색기 패널에서 전체 화면 아이콘 전체 화면을 선택하여 API 탐색기 창을 펼칩니다.
