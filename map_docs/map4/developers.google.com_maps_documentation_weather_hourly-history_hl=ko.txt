시간별 기록 요청 정보

시간별 기록 정보를 요청하려면 다음 주소로 HTTP GET 요청을 보내세요.

https://weather.googleapis.com/v1/history/hours:lookup?key=YOUR_API_KEY&location.latitude=LATITUDE&location.longitude=LONGITUDE

요청 URL 매개변수 [https://developers.google.com/maps/documentation/weather/reference/rest/v1/history.hours/lookup?hl=ko#query-parameters]에 위치의 위도 및 경도 좌표를 포함합니다.

시간별 기록 요청 예

기본적으로 history.hours [https://developers.google.com/maps/documentation/weather/reference/rest/v1/history.hours?hl=ko] 엔드포인트는 지난 시간부터 시작하여 24시간의 이전 날씨 데이터를 반환합니다. hours 매개변수를 사용하여 특정 기간으로 요청 범위를 지정할 수 있습니다.

다음 예에서는 캘리포니아주 마운틴뷰의 지난 3시간 동안의 시간별 기록 정보를 요청합니다.

curl -X GET "https://weather.googleapis.com/v1/history/hours:lookup?key=YOUR_API_KEY&location.latitude=37.4220&location.longitude=-122.0841&hours=3"

응답에서 interval 객체는 시간별 정보를 나타냅니다.

"historyHours": [
"interval": {
"startTime": "2025-02-12T22:00:00Z",
"endTime": "2025-02-12T23:00:00Z"

정보가 여러 페이지인 대답에는 nextPageToken가 포함됩니다. 정보의 다음 페이지를 보려면 요청에서 nextPageToken 값을 pageToken 매개변수에 전달하세요.
참고: 지정된 기간의 시간 중 하나라도 날씨 데이터를 사용할 수 없는 경우 토큰은 무효화됩니다. 잘못된 토큰을 pageToken에 전달하면 오류가 반환됩니다.
다음 예에서는 페이지당 5시간의 데이터로 캘리포니아주 마운틴 뷰의 8시간 분량 날씨 예보 데이터를 요청합니다.

curl -X GET "https://weather.googleapis.com/v1/history/hours:lookup?key=YOUR_API_KEY&location.latitude=37.4220&location.longitude=-122.0841&hours=8&pageSize=5"

처음 3시간의 데이터를 포함하는 응답은 다음과 같은 형식입니다.

"historyHours": [
"interval": {
"startTime": "2025-02-12T23:00:00Z",
"endTime": "2025-02-13T00:00:00Z"

페이지 오른쪽에서 API 아이콘 api을 선택합니다.
원하는 경우 요청 매개변수를 수정합니다.
실행 버튼을 선택합니다. 대화상자에서 요청을 보낼 때 사용할 계정을 선택합니다.
API 탐색기 패널에서 전체 화면 아이콘 전체 화면을 선택하여 API 탐색기 창을 펼칩니다.
