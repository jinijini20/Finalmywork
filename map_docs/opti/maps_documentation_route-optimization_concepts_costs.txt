비용 모델
Route Optimization API의 주요 목표는 비용이 가장 낮은 경로를 찾는 것입니다. 따라서 비용 모델은 경로 최적화의 주요 동인입니다.
비용 모델은 전역, 차량, 배송 비용을 지정하는 속성 집합입니다.
비용 모델 속성은 다음과 같은 종류의 최적화 목표를 지원합니다.
효율적인 차량 할당 및 경로
비용 효율적인 수령 및 배송 시간
중요한 배송의 우선순위 지정
구조
다이어그램에 표시된 것처럼 비용 모델 속성은 다음과 같이 구성됩니다.
Shipment [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/ShipmentModel?hl=ko#Shipment]에는 penaltyCost [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/ShipmentModel?hl=ko#Shipment.FIELDS.penalty_cost] 속성이 포함되어 있습니다.
Vehicle [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/ShipmentModel?hl=ko#Vehicle]에는 다음 속성이 포함됩니다.
fixedCost [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/ShipmentModel?hl=ko#Vehicle.FIELDS.fixed_cost]
costPerHour [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/ShipmentModel?hl=ko#Vehicle.FIELDS.cost_per_hour]
costPerKilometer [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/ShipmentModel?hl=ko#Vehicle.FIELDS.cost_per_kilometer]
costPerTraveledHour [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/ShipmentModel?hl=ko#Vehicle.FIELDS.cost_per_traveled_hour]
이 문서에서는 필수 비용 모델 매개변수만 강조합니다. 비용 매개변수의 전체 집합은 참조 문서를 확인하세요.
Essentials 체크리스트
다음 체크리스트에서는 잠재적인 비용 관련 실수를 방지하는 필수 지식을 설명합니다. 이 목록은 요청을 검증하고 응답 문제를 해결하는 데 도움이 됩니다.
최적화를 위해서는 비용이 필요합니다. 비용이 없는 요청은 API가 긴 대중교통을 포함한 비논리적인 경로를 반환하도록 합니다.
차량당 costPerHour [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/ShipmentModel?hl=ko#Vehicle.FIELDS.cost_per_hour], costPerKilometer [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/ShipmentModel?hl=ko#Vehicle.FIELDS.cost_per_kilometer], costPerTraveledHour [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/ShipmentModel?hl=ko#Vehicle.FIELDS.cost_per_traveled_hour] 중 하나 이상을 지정해야 합니다.
요청당 여러 비용 속성을 사용하여 더 정교한 최적화 목표를 달성할 수 있습니다.
비용 속성은 특정 통화와 연결되지 않습니다.
비용 단위가 나타내는 값을 결정하면 이 값이 요청의 모든 비용 속성에 적용됩니다.
속성
다음 표에는 비용 모델 속성이 나열되어 있습니다.
자녀 있음 속성 이름 숙소 유형 비용 - 기간: 숙박 시설 설명
Shipment penaltyCost [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/ShipmentModel?hl=ko#Shipment.FIELDS.penalty_cost] 숫자 배송 건너뜀 배송을 건너뛰는 데 드는 비용입니다.
완료 비용이 벌금 비용을 초과하면 API가 배송을 건너뜁니다.
penaltyCost [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/ShipmentModel?hl=ko#Shipment.FIELDS.penalty_cost]을 설정하면 비용이 높은 배송의 배송이 우선시됩니다.
배송 penaltyCost [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/ShipmentModel?hl=ko#Shipment.FIELDS.penalty_cost]을 설정하지 않으면 배송이 필수가 됩니다.
배송 penaltyCost [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/ShipmentModel?hl=ko#Shipment.FIELDS.penalty_cost]을 0으로 명시적으로 설정하면 다른 제약 조건이 필요하지 않는 한 API가 항상 배송을 건너뜁니다.
Vehicle fixedCost [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/ShipmentModel?hl=ko#Vehicle.FIELDS.fixed_cost] 숫자 배송 이 차량이 배송을 처리하는 데 사용되는 경우 적용되는 고정 비용입니다.
costPerHour [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/ShipmentModel?hl=ko#Vehicle.FIELDS.cost_per_hour] 숫자 시간 이동, 대기, 방문, 휴식 시간을 포함한 시간당 차량 운행 비용입니다.
이 비용이 증가하면 최적화 프로그램은 가장 짧은 경로가 아닐 수도 있는 더 빠른 경로를 찾으려고 시도합니다.
이 속성은 단순성과 완전성으로 인해 효과적인 독립형 차량당 비용이 될 수 있습니다.
costPerKilometer [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/ShipmentModel?hl=ko#Vehicle.FIELDS.cost_per_kilometer] 숫자 킬로미터 연료 비용, 상각된 차량 유지보수 비용 등 차량의 이동 거리당 비용입니다.
costPerTraveledHour [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/ShipmentModel?hl=ko#Vehicle.FIELDS.cost_per_traveled_hour] 숫자 시간 대기, 방문, 휴식 시간을 제외하고 이동 중에만 시간당 차량을 운행하는 비용입니다. 이 경우 더 짧은 경로보다 더 빠른 이동 경로가 우선시됩니다.

코드 샘플
다음 코드 샘플은 Shipment [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/ShipmentModel?hl=ko#Shipment]의 비용 모델 속성 구조를 보여줍니다.
{
  "model": {
    "shipments": [ ...
      {
        "penaltyCost": PENALTY_COST
      }
    ],
    "vehicles": [ ...
    ]
  }
}
다음 코드 샘플은 Vehicle [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/ShipmentModel?hl=ko#Vehicle]의 비용 모델 속성 구조를 보여줍니다.
{
  "model": {
    "shipments": [ ...
    ],
    "vehicles": [ ...
      {
        "fixedCost": FIXED_COST,
        "costPerKilometer": KILOMETER_COST,
        "costPerHour": HOUR_COST,
        "costPerTraveledHour": TRAVELED_HOUR_COST
      }
    ]
  }
}
예시 시나리오
이 섹션에서는 반려견 데이케어 비즈니스를 운영하는 시나리오를 설명합니다. 집에서 개를 데려오는 데 사용하는 트럭의 경로를 최적화하고 있습니다. 이 시나리오에서는 최적화 프로그램이 경로를 제공할 때 개 픽업 및 차량 운영과 관련된 비용을 고려해야 합니다.
이 예시에서 비용 단위 하나는 1달러를 나타냅니다. 즉, 요청의 비용 모델 속성 값은 다음과 같습니다.
속성 값 시나리오
penaltyCost [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/ShipmentModel?hl=ko#Shipment.FIELDS.penalty_cost] 10 예약된 날짜에 반려견을 데리러 오지 않은 고객에게 부과하는 벌금을 나타냅니다. 예정된 날짜에 강아지를 데리러 가지 않으면 고객의 서비스 총액에서 40달러가 할인됩니다.
fixedCost [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/ShipmentModel?hl=ko#Vehicle.FIELDS.fixed_cost] 30 차량 대출 납입금의 일일 비용을 나타냅니다. 일일 비용은 30달러입니다.
costPerKilometer [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/ShipmentModel?hl=ko#Vehicle.FIELDS.cost_per_kilometer] 0.08 차량의 킬로미터당 휘발유 소비량을 나타냅니다. 차량이 이동하는 데 필요한 연료는 1km당 0.04갤런이며, 해당 지역의 갤런당 비용은 2달러입니다.
costPerHour [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/ShipmentModel?hl=ko#Vehicle.FIELDS.cost_per_hour] 27 운전자가 차량을 운전한 데 대해 지불하는 금액을 나타냅니다. 시간당 27달러를 운전자에게 지불합니다.
costPerTraveledHour [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/ShipmentModel?hl=ko#Vehicle.FIELDS.cost_per_traveled_hour] 2.5 이동 중에 반려견을 위해 차량의 에어컨을 시간당 얼마나 지불해야 하는지를 나타냅니다. 차량이 움직이지 않을 때는 운전자가 뒷문을 열고 에어컨을 끌 수 있습니다.
비용 매개변수를 기반으로 최적화 프로그램은 사용자에게는 명확하지 않지만 최적화를 통해 찾을 수 있는 절충안을 만들 수 있습니다.
다음 다이어그램은 최적화 프로그램이 점선 빨간색의 트래픽을 피하기 위해 점선 녹색을 통해 더 길지만 더 빠른 경로를 선택할 수 있는 예를 보여줍니다.
이 시나리오에서 두 경로의 비용 균형은 다음과 같습니다.
녹색 점선은 교통 체증을 피하는 빠른 경로이므로 costPerHour [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/ShipmentModel?hl=ko#Vehicle.FIELDS.cost_per_hour] 및 costPerTraveledHour [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/ShipmentModel?hl=ko#Vehicle.FIELDS.cost_per_traveled_hour]이 낮습니다. 따라서 costPerKilometer [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/ShipmentModel?hl=ko#Vehicle.FIELDS.cost_per_kilometer]이 높더라도 비용 효율성이 더 높습니다.
빨간색 점선은 직접 경로이므로 costPerKilometer [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/ShipmentModel?hl=ko#Vehicle.FIELDS.cost_per_kilometer]이 낮지만 교통 체증으로 인한 대기 시간으로 인해 costPerHour [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/ShipmentModel?hl=ko#Vehicle.FIELDS.cost_per_hour] 및 costPerTraveledHour [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/ShipmentModel?hl=ko#Vehicle.FIELDS.cost_per_traveled_hour]이 너무 높아 가장 비싼 경로가 됩니다.
비용 효율적인 경로를 제공하는 것 외에도 최적화 도구는 응답 속성 [https://developers.google.com/maps/documentation/route-optimization/concepts/costs?hl=ko#response_properties]에서 배송 경로의 총비용 합계를 제공합니다.
요청 예
다음 예시에서는 예시 시나리오에 설정된 값을 통합하는 기본 optimizeTours [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/projects/optimizeTours?hl=ko] 요청의 구조를 보여줍니다.
{
  "model": {
    "shipments": [
      {
        "pickups": [
          {
            "arrivalLocation": {
              "latitude": 37.8024,
              "longitude": -122.4058
            }
          }
        ],
        "deliveries": [
          {
            "arrivalLocation": {
              "latitude": 37.759773,
              "longitude": -122.427063
            }
          }
        ]
        "penaltyCost": 40
      }
    ],
    "vehicles": [
      {
        "startLocation": {
          "latitude": 37.759773,
          "longitude": -122.427063
        },
        "endLocation": {
          "latitude": 37.759773,
          "longitude": -122.427063
        },
        "fixedCost": 30,
        "costPerKilometer": 0.08,
        "costPerHour": 27,
        "costPerTraveledHour": 2.5
      }
    ]
  }
}
응답 속성
OptimizeToursResponse [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/OptimizeToursResponse?hl=ko] 메시지에는 경로를 완료하는 과정에서 발생하는 비용을 설명하는 비용 속성이 있습니다.
metrics.costs [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/OptimizeToursResponse?hl=ko#Metrics.FIELDS.costs]: 비용 관련 요청 필드별로 분류된 모든 경로의 총비용입니다.
metrics.totalCost [https://developers.google.com/maps/documentation/route-optimization/reference/rest/v1/OptimizeToursResponse?hl=ko#Metrics.FIELDS.total_cost]: 모든 경로의 총비용을 합산한 값입니다.
