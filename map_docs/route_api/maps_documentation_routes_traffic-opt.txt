사용 가능한 트래픽 옵션
유럽 경제 지역 (EEA) 개발자
경로를 요청할 때 경로를 계산할 때 Routes API가 교통 데이터를 사용하는 방법을 선택할 수 있습니다. 교통 모델은 이전 및 실시간 교통 상황을 경로 생성에 반영하는 방법으로, 표시되는 경로와 예상 이동 시간에 모두 영향을 줄 수 있습니다.
다음은 경로의 교통 옵션입니다.
트래픽 옵션 설명
트래픽 데이터를 포함할지 여부와 방법을 지정합니다 [https://developers.google.com/maps/documentation/routes/config_trade_offs?hl=ko] 더 정확한 결과를 얻으려면 트래픽 데이터를 포함하고, 더 빠른 응답을 원하면 트래픽 데이터를 제외하세요.
다중선에 교통정보 반영 여부 선택 [https://developers.google.com/maps/documentation/routes/traffic_on_polylines?hl=ko#request_a_traffic-aware_polyline] 반환된 폴리라인에 경로의 교통 상황에 관한 정보를 포함할 수 있습니다. 이 기능은 더 높은 요율로 청구됩니다.
사용할 트래픽 모델 지정 [https://developers.google.com/maps/documentation/routes/traffic-model?hl=ko] 교통 상황에서 시간을 계산하는 데 사용되는 다양한 요소를 우선순위로 지정하는 데 사용할 교통 모델을 지정합니다.
