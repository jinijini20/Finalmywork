경로 매트릭스의 위치 지정
유럽 경제 지역 (EEA) 개발자
경로 매트릭스를 계산하려면 최소한 하나의 경로 출발지와 하나의 경로 목적지의 위치를 지정해야 합니다.
경로의 위치 지정
경유지 (REST) [https://developers.google.com/maps/documentation/routes/reference/rest/v2/Waypoint?hl=ko] 또는 경유지 (gRPC) [https://developers.google.com/maps/documentation/routes/reference/rpc/google.maps.routing.v2?hl=ko#waypoint] 객체를 만들어 위치를 나타냅니다. 다음 방법 중 하나로 위치를 지정할 수 있습니다.
장소 ID [https://developers.google.com/maps/documentation/routes/specify_location-rm?hl=ko#place_id] (권장)
위도/경도 좌표 [https://developers.google.com/maps/documentation/routes/specify_location-rm?hl=ko#lat_long]
주소 문자열 [https://developers.google.com/maps/documentation/routes/specify_location-rm?hl=ko#text_string] ('시카고, 일리노이' 또는 '다윈, NT, 오스트레일리아')
Plus Code [https://developers.google.com/maps/documentation/routes/specify_location-rm?hl=ko#plus_code]
요청에서 모든 웨이포인트의 위치를 동일한 방식으로 지정하거나 혼합할 수 있습니다. 예를 들어 출발 웨이포인트에는 위도/경도 좌표를 사용하고 목적지 웨이포인트에는 장소 ID를 사용할 수 있습니다.
효율성과 정확성을 위해 위도/경도 좌표 또는 주소 문자열 대신 장소 ID를 사용하세요. 장소 ID는 고유하게 명시적이며 액세스 포인트 및 트래픽 변수와 같은 경로 설정에 지오코딩 이점을 제공합니다. 위치를 지정하는 다른 방법으로 인해 발생할 수 있는 다음과 같은 상황을 방지하는 데 도움이 됩니다.
위도/경도 좌표를 사용하면 해당 좌표에 가장 가까운 도로에 위치가 스냅될 수 있습니다. 이 도로는 숙박 시설의 진입로가 아닐 수도 있고, 목적지로 빠르거나 안전하게 연결되는 도로가 아닐 수도 있습니다.
경로를 계산하려면 먼저 Routes API로 주소 문자열을 지오코딩하여 위도/경도 좌표로 변환해야 합니다. 이 전환은 실적에 영향을 미칠 수 있습니다.
위치를 장소 ID로 지정
장소 ID를 사용하여 중간 지점의 위치를 지정할 수 있습니다. 위도와 경도 좌표가 도로에 스냅되므로 경우에 따라 장소 ID가 더 나은 결과를 제공할 수 있습니다.
Geocoding API [https://developers.google.com/maps/documentation/geocoding?hl=ko] 및 Places API [https://developers.google.com/maps/documentation/places/web-?hl=ko] (Place Autocomplete 포함)에서 장소 ID를 가져옵니다. 장소 ID에 관한 자세한 내용은 장소 ID 개요 [https://developers.google.com/maps/documentation/places/web-service/place-id?hl=ko]를 참고하세요.
다음 예에서는 placeId 속성을 사용하여 origin 및 destination 모두의 장소 ID를 전달합니다.
{
  "origin":{
    "placeId": "ChIJayOTViHY5okRRoq2kGnGg8o"
  },
  "destination":{
    "placeId": "ChIJTYKK2G3X5okRgP7BZvPQ2FU"
  },
  ...
}
위치를 위도 및 경도 좌표로 지정
경유지에서 위치를 정의하려면 위도/경도 좌표를 사용하여 위치 (REST) [https://developers.google.com/maps/documentation/routes/reference/rest/v2/Location?hl=ko] 또는 위치(gRPC) [https://developers.google.com/maps/documentation/routes/reference/rpc/google.maps.routing.v2?hl=ko#location]를 지정합니다.
예를 들어 latitude 및 longitude 좌표를 사용하여 경로 origin 및 destination의 중간 지점을 지정합니다.
{
  "origin":{
    "location":{
      "latLng":{
        "latitude": 37.419734,
        "longitude": -122.0827784
      }
    }
  },
  "destination":{
    "location":{
      "latLng":{
        "latitude": 37.417670,
        "longitude": -122.079595
      }
    }
  },
...
}
참고: 위도/경도 좌표로 지정된 지점은 도로에 스냅되므로 앱에 필요한 정확성을 제공하지 않을 수 있습니다. 가능한 액세스 포인트나 추가 지오코딩 세부정보와 관계없이 앱에서 경로 설정에 필요한 지점을 실제로 지정하는 값이라고 확신하는 경우 위도/경도 좌표를 사용하세요.
위치를 주소 문자열로 지정
주소 문자열은 문자열로 표현된 리터럴 주소입니다 (예: '1600 Amphitheatre Parkway, Mountain View, CA'). 지오코딩은 주소 문자열을 위도 및 경도 좌표 (예: 위도 37.423021 및 경도 -122.083739)로 변환하는 과정입니다.
주소 문자열을 중간 지점의 위치로 전달하면 Routes API는 내부적으로 문자열을 지오코딩하여 위도와 경도 좌표로 변환합니다.
참고: 위도 및 경도 좌표는 Geocoding API [https://developers.google.com/maps/documentation/geocoding?hl=ko]에서 반환한 좌표와 다를 수 있습니다. 예를 들어 Routes API는 건물 중앙이 아닌 건물 입구의 좌표를 반환할 수 있습니다.
예를 들어 경로를 계산하려면 주소 문자열을 사용하여 경로 origin 및 destination의 중간 지점을 지정합니다.
{
  "origin":{
    "address": "1600 Amphitheatre Parkway, Mountain View, CA"
  },
  "destination":{
    "address": "450 Serra Mall, Stanford, CA 94305, USA"
  },
  ...
}
이 예에서 Routes API는 두 주소를 지오코딩하여 위도 및 경도 좌표로 변환합니다.
주소 값이 모호한 경우 Routes API는 유사한 주소와 구분하기 위해 검색을 호출할 수 있습니다. 예를 들어 '1st Street'는 '1st street NE' 또는 '1st St SE'의 전체 값 또는 일부 값일 수 있습니다. 이 결과는 Geocoding API에서 반환한 결과와 다를 수 있습니다. 장소 ID를 사용하면 오해의 소지를 방지할 수 있습니다.
주소의 리전 설정
불완전한 주소 문자열을 중간 지점의 위치로 전달하면 API에서 잘못된 지오코딩된 위도/경도 좌표를 사용할 수 있습니다. 예를 들어 운전 경로의 출발지로 '톨레도'를, 목적지로 '마드리드'를 지정하는 요청을 실행합니다.
{
  "origin":{
    "address": "Toledo"
  },
  "destination":{
    "address": "Madrid"
  },
  "travelMode": "DRIVE"
}
이 예에서 'Toledo'는 스페인이 아닌 미국 오하이오주의 도시로 해석됩니다. 따라서 요청은 빈 배열을 반환합니다. 즉, 경로가 없음을 의미합니다.
{
  []
}
regionCode 매개변수를 포함하여 특정 지역에 편중된 결과를 반환하도록 API를 구성할 수 있습니다. 이 매개변수는 지역 코드를 ccTLD ('최상위 도메인') [https://en.wikipedia.org/wiki/List_of_Internet_top-level_domains#Country_code_top-level_domains] 2자리 값으로 지정합니다. 대부분의 ccTLD 코드는 ISO 3166-1 코드와 동일하지만 일부 주목할 만한 예외가 있습니다. 예를 들어 영국의 ccTLD는 'uk'(.co.uk)이지만 ISO 3166-1 코드는 'gb'입니다(기술적으로 '그레이트 브리튼 북아일랜드 연합왕국'의 항목에 해당).
regionCode 매개변수가 포함된 '톨레도'에서 '마드리드'까지의 경로 요청은 '톨레도'가 스페인의 도시로 해석되므로 적절한 결과를 반환합니다.
{
  "origin":{
    "address": "Toledo"
  },
  "destination":{
    "address": "Madrid"
  },
  "travelMode": "DRIVE",
  "regionCode": "es"
}
이제 응답에 스페인 토레도에서 스페인 마드리드까지 계산된 경로가 포함됩니다.
{
  "routes": [
    {
      "distanceMeters": 75330,
      "duration": "4137s",
      ...
    }
  ]
}
위치를 플러스 코드로 지정
정확한 주소가 없는 경우가 많아 배송받기가 어려울 수 있습니다. 또는 주소가 있는 사용자가 후문이나 하역장과 같이 더 구체적인 위치에서 배송을 받기를 원할 수도 있습니다.
플러스 코드는 실제 주소가 없는 사람 또는 장소의 상세 주소와 같습니다. 플러스 코드는 도로 이름과 번호가 포함된 주소 대신 위도/경도 좌표를 기반으로 하며 숫자와 문자로 표시됩니다.
Google은 모든 사람과 사물에 주소의 이점을 제공하기 위해 Plus Code [https://maps.google.com/pluscodes/?hl=ko]를 개발했습니다. Plus Code는 위도/경도 좌표에서 파생된 인코딩된 위치 참조로 1/8000도x1/8000도 (적도에서 14mx14m) 이하의 영역을 나타냅니다. Plus Code는 상세 주소가 없는 장소 또는 건물에 번호가 지정되지 않거나 거리 이름이 없는 장소의 상세 주소 대신 사용할 수 있습니다.
Plus Code는 글로벌 코드 또는 복합 코드로 형식이 지정되어야 합니다.
글로벌 코드는 4자리 지역 번호와 6자 이상의 로컬 코드로 구성됩니다.
예를 들어 '1600 Amphitheatre Parkway, Mountain View, CA' 주소의 경우 글로벌 코드는 '849V'이고 로컬 코드는 'CWC8+R9'입니다. 그런 다음 전체 10자리 Plus Code를 사용하여 위치 값을 '849VCWC8+R9'로 지정합니다.
복합 코드는 명시적인 위치와 결합된 6자 이상의 로컬 코드로 구성됩니다.
예를 들어 '450 Serra Mall, Stanford, CA 94305, USA' 주소의 로컬 코드는 'CRHJ+C3'입니다. 복합 주소의 경우 'CRHJ+C3 Stanford, CA 94305, USA'와 같은 형식으로 로컬 코드를 주소의 도시, 주, 우편번호, 국가 부분과 결합합니다.
예를 들어 Plus 코드를 사용하여 경로 origin 및 destination의 중간 지점을 지정하여 경로를 계산합니다.
{
  "origin":{
    "address": "849VCWC8+R9"
  },
  "destination":{
    "address": "CRHJ+C3 Stanford, CA 94305, USA"
  },
  "travelMode": "DRIVE"
}
Plus 코드는 장소 자동 완성 [https://developers.google.com/maps/documentation/places/web-service/autocomplete?hl=ko], 장소 세부정보 [https://developers.google.com/maps/documentation/places/web-service/details?hl=ko], Directions API (기존) [https://developers.google.com/maps/documentation/directions?hl=ko], Geocoding API [https://developers.google.com/maps/documentation/geocoding?hl=ko]를 비롯한 Google Maps Platform API에서 지원됩니다. 예를 들어 Geocoding API를 사용하여 위도/경도 좌표로 지정된 위치를 역 지오코딩하여 위치의 Plus Code를 확인할 수 있습니다.
