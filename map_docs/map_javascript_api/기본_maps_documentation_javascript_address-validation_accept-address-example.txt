주소 수락 - 예시
이 제품 또는 기능은 미리보기(GA 이전) 버전입니다. GA 이전 제품과 기능은 지원이 제한적일 수 있으며, GA 이전 제품과 기능에 대한 변경사항은 다른 GA 이전 버전과 호환되지 않을 수 있습니다. GA 이전 버전 제품 및 서비스에는 Google Maps Platform 서비스별 약관 [https://cloud.google.com/maps-platform/terms/maps-service-terms?hl=ko]이 적용됩니다. 자세한 내용은 출시 단계 설명 [https://developers.google.com/maps/launch-stages?hl=ko]을 참고하세요.
이 문서에서는 주소 유효성 검사 API가 시스템에서 수락 동작을 보장하는 응답 신호를 제공하는 여러 실제 시나리오를 설명합니다. 컨텍스트는 유효성 검사 응답 사용의 워크플로 개요 [https://developers.google.com/maps/documentation/javascript/address-validation/use-validation-response?hl=ko#workflow-overview]를 참고하세요.
참고: 여기에 나온 예는 일부에 해당합니다.
일반적인 예: 수락
이 시나리오에서는 시스템이 고객이 입력한 주소를 수락하는 주소를 보여줍니다.
주소 입력됨 지역
76 Buckingham Palace Road, London SW1W 9TQ 영국
허용되는 주소에 대한 평결
아래 예에서는 중요한 신호를 강조 표시합니다.
{
  "inputGranularity": "PREMISE",
  "validationGranularity": "PREMISE",
  "geocodeGranularity": "PREMISE",
  "addressComplete": true
}
이 외에도 verdict [https://developers.google.com/maps/documentation/javascript/reference/address-validation?hl=ko#Verdict]은 다음을 나타냅니다.
hasUnconfirmedComponents는 false로 남아 있음
hasInferredComponents는 false로 남아 있음
hasReplacedComponents는 false로 남아 있음
이러한 신호를 함께 사용하면 고품질 주소를 나타냅니다.
작업: 수락합니다.
특이 사례 예: 수락
다음 예에서는 추가 조사가 필요한 주소 품질 문제를 verdict [https://developers.google.com/maps/documentation/javascript/reference/address-validation?hl=ko#Verdict]가 나타내는 상황을 다룹니다. 이 예에서는 시스템 로직을 개선하기 위해 평결에서 주소 구성요소로 로직을 이동하여 더 완전한 그림을 얻는 방법을 보여줍니다.
참고: 이 예는 일부에 해당하지만 대표적인 예입니다.
미국 외 지역의 확인되지 않은 번지
이 예시에서는 추론되거나 대체된 구성요소 없이 모든 주소 구성요소가 있는 이탈리아 주소의 입력을 보여줍니다. 하지만 validationGranularity [https://developers.google.com/maps/documentation/javascript/reference/address-validation?hl=ko#Verdict.validationGranularity]은 ROUTE입니다.
주소 입력됨 지역
Via Fonte Grugnale, 14 unit 2, 66054 Vasto CH, Italia IT
확인되지 않은 번지에 대한 평결
{
  "inputGranularity": "SUB_PREMISE",
  "validationGranularity": "ROUTE",
  "geocodeGranularity": "ROUTE",
  "addressComplete": true,
  "hasUnconfirmedComponents": true
}
주소 구성요소를 추가로 조사한 결과, 번지의 확인 수준 [https://developers.google.com/maps/documentation/javascript/reference/address-validation?hl=ko#ConfirmationLevel]이 UNCONFIRMED_BUT_PLAUSIBLE인 것으로 확인되었습니다.
{
   "text": "14",
   "componentType": "street_number",
   "confirmationLevel": "UNCONFIRMED_BUT_PLAUSIBLE"
}
작업: 이 주소는 추가 메시지 없이 수락할 수 있습니다. 하지만 신뢰도 수준을 높이려면 고객에게 주소를 확인하라는 메시지를 표시할 수 있습니다. 자세한 내용은 유효성 검사 로직 빌드의 구현 안내 [https://developers.google.com/maps/documentation/javascript/address-validation/build-validation-logic?hl=ko#implementation_guidance]를 참고하세요.
미국에서 확인되지 않은 번지
이 예시에서는 추론되거나 대체된 구성요소 없이 모든 주소 구성요소가 있는 미국 주소의 입력을 보여줍니다. 하지만 validationGranularity [https://developers.google.com/maps/documentation/javascript/reference/address-validation?hl=ko#Verdict.validationGranularity]은 PREMISE_PROXIMITY입니다.
주소 입력됨 지역
975 Carson Dr, Sunnyvale, CA 94086 미국
확인되지 않은 번지에 대한 USPS 데이터
{
   "firstAddressLine": "975 CARSON DR",
   "cityStateZipAddressLine": "SUNNYVALE CA 94086",
   "city": "SUNNYVALE",
   "state": "CA",
   "zipCode": "94086"
   "dpvConfirmation": "N",
   "dpvFootnote": "AAM3",
   "carrierRoute": "C031",
   "carrierRouteIndicator": "D",
   "postOfficeCity": "SUNNYVALE",
   "postOfficeState": "CA",
   "fipsCountyCode": "085",
   "county": "SANTA CLARA",
}
작업: 수정 이 예시는 이전 예시와 비슷하며 주소는 입력된 대로 허용될 수 있습니다. 하지만 N의 추가 dpvConfirmation 코드는 고객에게 주소를 확인하거나 주소를 수정하라는 메시지를 표시하는 강력한 신호를 제공합니다. 이 경우에 선택하는 동작은 궁극적으로 필요한 신뢰도 수준에 따라 달라집니다. 자세한 내용은 유효성 검사 로직 빌드의 구현 안내 [https://developers.google.com/maps/documentation/javascript/address-validation/build-validation-logic?hl=ko#implementation_guidance]를 참고하세요.
확인된 주소의 USPS 데이터가 불완전함
이 예에서는 모든 주소 구성요소가 확인되고 추론되거나 대체된 구성요소가 없으며 PREMISE의 validationGranularity [https://developers.google.com/maps/documentation/javascript/reference/address-validation?hl=ko#Verdict.validationGranularity]이 있는 미국 주소의 입력을 보여줍니다. 하지만 uspsData [https://developers.google.com/maps/documentation/javascript/reference/address-validation?hl=ko#USPSData]이 완전히 채워지지 않고 dpvConfirmation 값이 포함되어 있지 않습니다.
주소 입력됨 지역
155 Via Condado Way, Palm Beach Gardens, FL 33418-1703 미국
USPS 데이터가 불완전한 확인된 주소에 대한 평결
{
  "inputGranularity": "PREMISE",
  "validationGranularity": "PREMISE",
  "geocodeGranularity": "PREMISE",
  "addressComplete": true,
}
USPS 데이터가 불완전한 확인된 주소의 USPS 데이터
{
  "standardizedAddress": {
    "firstAddressLine": "155 VIA CONDADO WAY",
    "cityStateZipAddressLine": "PALM BEACH GARDENS",
    "city": "PALM BEACH GARDENS",
    "state": "FL",
    "zipCode": "33418"
  },
  "carrierRoute": "H018",
  "postOfficeCity": "PALM BEACH GARDENS",
  "postOfficeState": "FL"
}
작업: 수락합니다. 주소가 USPS에서 DPV 확인을 거치지 않았지만 PREMISE의 validationGranularity는 이 주소가 유효하다는 강력한 신호를 제공합니다. 하지만 신뢰도 수준을 높이려면 고객에게 주소를 확인하라는 메시지를 표시할 수 있습니다. 자세한 내용은 유효성 검사 로직 빌드의 구현 안내 [https://developers.google.com/maps/documentation/javascript/address-validation/build-validation-logic?hl=ko#implementation_guidance]를 참고하세요.
참고: 서비스는 업데이트된 USPS 데이터를 주기적으로 수신하므로 이 예시에서는 더 이상 불완전한 USPS 데이터를 반환하지 않을 수 있습니다. 이 예는 이 주소의 API 응답을 정확하게 묘사하기보다는 설명하기 위한 것입니다.
