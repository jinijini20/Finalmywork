시작하기 및 설정
플랫폼 선택: Android [https://developers.google.com/maps/documentation/android-sdk/cloud-customization/setup-leg?hl=ko] iOS [https://developers.google.com/maps/documentation/ios-sdk/cloud-customization/setup-leg?hl=ko] JavaScript [https://developers.google.com/maps/documentation/javascript/cloud-customization/setup-leg?hl=ko] 웹 서비스 [https://developers.google.com/maps/documentation/maps-static/cloud-customization/setup-leg?hl=ko]
Android에서 클라우드 기반 지도 스타일 지정을 사용하려면 Android용 Maps SDK를 위한 최신 렌더기를 사용해야 합니다. 다음 안내를 참고하세요 [https://developers.google.com/maps/documentation/android-sdk/renderer?hl=ko].
클라우드 기반 지도 스타일 지정의 최신 버전에는 기존 스타일 지정보다 더 많은 지도 지형지물과 스타일 지정 옵션 및 새 지도 스타일로 업데이트하거나 이를 사용하기 전에 알아야 하는 변경사항이 포함되어 있습니다. 이 버전 사용에 대한 자세한 내용은 알려진 문제 [https://developers.google.com/maps/documentation/android-sdk/cloud-customization/troubleshoot?hl=ko#known_issues] 및 다음 섹션을 참고하세요.
요금
클라우드 기반 지도 스타일 지정을 사용하려면 지도 ID [https://developers.google.com/maps/documentation/get-map-id?hl=ko]가 필요합니다. Android용 Maps SDK, iOS용 Maps SDK, JavaScript에서 지도 ID를 사용하면 Dynamic Maps SKU [https://developers.google.com/maps/billing-and-pricing/pricing?hl=ko#dynamic-maps]에 대한 요금이 부과됩니다. Maps Static API에서 지도 ID를 사용하면 Static Maps [https://developers.google.com/maps/billing-and-pricing/pricing?hl=ko#static-maps] SKU에 대한 요금이 부과됩니다.
애플리케이션 및 웹사이트 요구사항
지도 스타일을 표시하려면 웹사이트 및 애플리케이션에 다음 플랫폼 버전이 필요합니다.
JavaScript 벡터 기반 지도: JavaScript Maps API 3.47 이상. 최신 버전을 설정하려면 Maps JavaScript API 설정 [https://developers.google.com/maps/documentation/javascript/load-maps-js-api?hl=ko]을 참고하세요.
JavaScript 래스터 기반 지도: JavaScript Maps API 3.49 이상. 최신 버전을 설정하려면 Maps JavaScript API 설정 [https://developers.google.com/maps/documentation/javascript/load-maps-js-api?hl=ko]을 참고하세요.
iOS: iOS Maps SDK 8.3.1 이상. 최신 버전을 설정하려면 Xcode 프로젝트 설정 [https://developers.google.com/maps/documentation/ios-sdk/config?hl=ko]을 참고하세요.
Android: Android SDK 버전 18.0.0 이상, 새로운 Android 지도 렌더기 [https://developers.google.com/maps/documentation/android-sdk/renderer?hl=ko] 및 GMSCore 컨테이너 버전 23.33.16. 최신 버전을 설정하려면 Android 스튜디오 프로젝트 설정 [https://developers.google.com/maps/documentation/android-sdk/config?hl=ko]을 참고하세요.
참고: JavaScript 지도에서 대부분의 클라우드 스타일 지정 기능을 사용하려면 Maps JavaScript API 베타 채널 [https://developers.google.com/maps/documentation/javascript/versions?hl=ko#choosing-the-beta-channel]을 사용해 보세요.
시작할 준비가 되셨나요? 튜토리얼 [https://developers.google.com/maps/documentation/android-sdk/cloud-customization/tut-leg?hl=ko]을 참고하세요.
기존 스타일 업데이트
순서와 계층이 변경되었기 때문에 이 버전을 사용하려면 현재의 스타일이나 프로세스를 변경해야 할 수 있습니다. 예를 들어 수역 스타일을 지정하려면, 자연 > 수역 섹션을 확인합니다. 새 계층을 모두 보려면 지도에서 스타일을 지정할 수 있는 항목 [https://developers.google.com/maps/documentation/android-sdk/cloud-customization/taxonomy?hl=ko#hier]을 참고하세요.
새로운 지형지물을 사용하려면 기존 지도 스타일을 업데이트하거나 다시 생성해야 합니다. 현재의 계층을 이 버전과 비교하려면 지도 지형지물 변경사항 [https://developers.google.com/maps/documentation/android-sdk/cloud-customization/update-old2-leg?hl=ko]을 참고하세요.
기존 지도 스타일을 업데이트해야 하는 경우 최신 버전으로 업데이트 [https://developers.google.com/maps/documentation/android-sdk/cloud-customization/update-legacy?hl=ko]를 참고하세요.
