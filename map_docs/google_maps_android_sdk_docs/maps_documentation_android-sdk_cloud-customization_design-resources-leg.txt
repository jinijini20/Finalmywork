디자인 체크리스트
플랫폼 선택: Android [https://developers.google.com/maps/documentation/android-sdk/cloud-customization/design-resources-leg?hl=ko] iOS [https://developers.google.com/maps/documentation/ios-sdk/cloud-customization/design-resources-leg?hl=ko] JavaScript [https://developers.google.com/maps/documentation/javascript/cloud-customization/design-resources-leg?hl=ko] 웹 서비스 [https://developers.google.com/maps/documentation/maps-static/cloud-customization/design-resources-leg?hl=ko]
다음 가이드라인에서는 지도 스타일을 만들거나 수정할 때 디자인상의 일반적인 문제점에 대해 다룹니다. 그래픽 디자이너에게 문의하여 디자인을 조합하고 검토하는 것이 좋습니다.
색상 대비가 충분한지 확인하기: 대비는 텍스트 가독성을 유지하고 특정 지도 지형지물을 구별하는 데 도움이 됩니다. 자세한 내용은 대비되는 채우기 및 획 색상 선택하기 [https://developers.google.com/maps/documentation/android-sdk/cloud-customization/design-resources-leg?hl=ko#contrast]를 참고하세요.
다양한 확대/축소 수준 확인하기: 스타일 요소가 다양한 확대/축소 수준에서 적절하게 표시되는지 테스트합니다.
색맹 사용자에게 문제가 되는 색상 조합 피하기: 색맹 사용자가 구별 가능한 디자인인지 확인합니다.
야간 모드 확인하기: 모바일 디자인의 경우 야간 모드에서 잘 작동하는지 확인합니다.
대비되는 채우기 및 획 색상 선택하기
채우기 및 획 색상이 충분히 대비되어 지도 지형지물이 배경 및 유사한 지도 지형지물과 구별될 수 있는지 확인합니다.
선의 다각형: 선에 세부사항을 표시하려면 대비되는 색상을 선택합니다. 예를 들어 트레일은 점선이지만, 색상 대비가 충분하지 않으면 실선처럼 보입니다. 고속도로는 외부 가장자리에 윤곽선이 표시되기 때문에 대비되지 않으면 단색으로 보일 수 있으므로 다른 도로 유형과 혼동될 수 있습니다.
도형의 다각형: 대비되는 색상을 사용하면 나란히 놓인 다각형 사이의 경계선을 표시하는 데 도움이 됩니다.
텍스트 획 및 채우기: 텍스트에서 획이란 윤곽선 색상을 말하며, 배경 색상에 대비되어 텍스트가 잘 보이도록 합니다. 획 색상이 채우기 색상과 너무 비슷할 경우 글꼴이 흐릿하게 보이거나 초점이 맞지 않을 수도 있습니다.
