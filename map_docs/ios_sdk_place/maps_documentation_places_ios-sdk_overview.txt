유럽 경제 지역 (EEA) 개발자
시작하기 전에: iOS용 Places SDK를 사용하려면 결제 계정이 있고 Places API 사용 설정된 프로젝트가 필요합니다. 프로젝트 소유자 및 결제 관리자를 여러 명 만들어 이러한 역할을 맡은 담당자가 팀에 상주하도록 하는 것이 좋습니다. 자세한 내용은 Cloud 콘솔에서 설정하기를 참고하세요.
소개
iOS용 Places SDK를 사용하면 기기 근처의 지역 비즈니스 및 기타 장소에 상황에 맞게 응답하는 위치 인식 앱을 빌드할 수 있습니다. 즉, 사용자에게 의미 있는 장소를 기반으로 풍부한 앱을 빌드할 수 있습니다.
참고: Places API (New)는 현재 버전의 Places SDK for iOS입니다. Places API가 기존 API로 전환되어 더 이상 사용 설정할 수 없습니다. 기존 문서 및 이전 가이드는 기존 탭에서 확인할 수 있습니다.
개념
장소는 이름이 있는 실제 공간으로 정의됩니다. 장소는 지도에서 찾을 수 있는 모든 것이라고 생각할 수도 있습니다. 예를 들어 지역 비즈니스, 관심 장소, 지리적 위치 등이 있습니다. API에서 장소는 GMSPlace Class Reference로 표현됩니다. 여기에는 장소 이름, 주소, 지리적 위치, 장소 ID, 전화번호, 장소 유형, 웹사이트 URL 등의 정보가 포함됩니다.
참고: 앱에서 iOS용 Places SDK에서 제공된 장소 정보를 표시할 때마다 API에서 반환된 모든 관련 저작자 표시도 함께 표시해야 합니다. 기여 표시에 관한 문서를 참고하세요.
SDK 개요
iOS용 Places SDK를 사용하여 고객이 현재 위치와 주변 정보를 탐색하도록 지원하세요. iOS용 Places SDK는 다음 API를 제공합니다.
iOS용 Places SDK (신규) 설명
Place Details (신규) 장소의 이름과 주소, 지리적 위치, 장소 유형 (예: 나이트클럽, 애완동물 가게, 박물관) 등 장소에 관한 정보를 반환합니다.
장소 사진 (신규) 장소의 고화질 이미지를 반환합니다.
텍스트 검색 (신규) 텍스트 검색을 기반으로 일련의 장소에 대한 정보를 반환합니다.
주변 지역 검색 (신규) 위치를 검색 영역으로 지정하여 일련의 장소에 대한 정보를 반환합니다.
자동 완성 (신규) 사용자가 입력할 때 장소의 이름 또는 주소를 자동으로 입력합니다.
SDK의 기타 주요 기능은 다음과 같습니다.
Wi-Fi 검색을 통해 전력이 낮은 상태에서 정확한 장소 감지

정책 및 약관
iOS용 Places SDK를 사용하는 모든 애플리케이션은 Google Maps Platform 서비스 약관 및 저작자 표시에 설명된 요구사항을 준수해야 합니다.
