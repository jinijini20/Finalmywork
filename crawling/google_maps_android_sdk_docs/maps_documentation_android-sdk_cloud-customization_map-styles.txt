지도 스타일 만들기 및 사용
플랫폼 선택: Android [https://developers.google.com/maps/documentation/android-sdk/cloud-customization/map-styles-leg?hl=ko] iOS [https://developers.google.com/maps/documentation/ios-sdk/cloud-customization/map-styles-leg?hl=ko] JavaScript [https://developers.google.com/maps/documentation/javascript/cloud-customization/map-styles-leg?hl=ko] 웹 서비스 [https://developers.google.com/maps/documentation/maps-static/cloud-customization/map-styles-leg?hl=ko]
Android에서 클라우드 기반 지도 스타일 지정을 사용하려면 Android용 Maps SDK를 위한 최신 렌더기를 사용해야 합니다. 다음 안내를 참고하세요 [https://developers.google.com/maps/documentation/android-sdk/renderer?hl=ko].
지도 스타일은 지도 지형지물을 시각적으로 맞춤설정한 것으로, 지도 ID와 연결됩니다. 그런 다음 애플리케이션 코드에서 지도 ID를 참조하여 맞춤설정된 지도를 표시합니다.
지도 ID를 만들기 전에 지도 스타일을 만들 수 있습니다. 그러나 스타일을 사용하려면 스타일을 지도 ID와 연결해야 합니다. 지도 스타일을 사용하려면 다음 단계를 모두 수행해야 합니다.
지도 스타일을 만들고 게시합니다. 자세한 내용은 지도 만들기 [https://developers.google.com/maps/documentation/android-sdk/cloud-customization/map-styles-leg?hl=ko#create-style] 및 스타일 게시 [https://developers.google.com/maps/documentation/android-sdk/cloud-customization/map-styles-leg?hl=ko#publish-style]를 참고하세요.
지도 ID를 만듭니다. 자세한 내용은 지도 ID 만들기 [https://developers.google.com/maps/documentation/get-map-id?hl=ko#create-a-map-id]를 참고하세요.
지도 스타일을 지도 ID와 연결합니다. 자세한 내용은 지도 ID를 지도 스타일에 연결 [https://developers.google.com/maps/documentation/get-map-id?hl=ko#associate-map-id-to-map-style]을 참고하세요.
앱에 지도 ID를 추가하여 스타일을 표시합니다. 자세한 내용은 앱에 지도 ID 추가하기 [https://developers.google.com/maps/documentation/get-map-id?hl=ko#add-a-map-id-to-your-app]를 참고하세요.
팁: 지도 ID가 애플리케이션 코드에 있으면 애플리케이션 코드를 변경하지 않고도 지도 스타일을 변경할 수 있으며, 최종 사용자는 업그레이드하지 않고도 업데이트된 지도 스타일을 볼 수 있습니다. 대신 지도 ID와 연결된 지도 스타일을 편집하면 변경사항이 애플리케이션 지도에 표시됩니다.
지도 스타일 만들기
다음 방법 중 하나를 사용하여 지도 스타일을 만듭니다.
기본 Google 지도 스타일을 사용하여 새로운 스타일을 만듭니다 [https://developers.google.com/maps/documentation/android-sdk/cloud-customization/map-styles-leg?hl=ko#create-new-style].
기존 표준 스타일 또는 맞춤설정된 스타일을 복제 [https://developers.google.com/maps/documentation/android-sdk/cloud-customization/map-styles-leg?hl=ko#duplicate-style]하고 수정합니다.
새 스타일 만들기
Google Cloud 콘솔에서 지도 스타일 [https://console.cloud.google.com/google/maps-apis/studio/styles?hl=ko]로 이동하고, 필요한 경우 프로젝트를 선택합니다.
지도 스타일에서 스타일 만들기를 선택합니다.
하단에서 스타일 편집기에서 열기를 선택합니다.
지도 지형지물을 선택하고 스타일을 변경하여 원하는 지도 스타일을 만듭니다. 맞춤 스타일이 있는 지도 지형지물 옆에는 파란색 단색 점이 표시됩니다.
참고: 모든 스타일이 모든 확대/축소 수준에서 나타나는 것은 아닙니다.
저장을 선택합니다.
지도 스타일 저장 및 게시 상자에 스타일 이름과 원하는 경우 설명을 입력하고 저장을 선택합니다.
스타일이 자동으로 게시되고 새 스타일의 기본 페이지가 표시됩니다.
주의: 스타일을 변경했지만 명시적으로 저장하지 않으면 지도 스타일의 버전 기록에 저장되지 않은 변경사항이 생성됩니다. 이러한 버전은 브라우저 탭을 닫거나 Cloud 콘솔 세션이 종료되면 사라집니다.
스타일 복제
지도 스타일 [https://console.cloud.google.com/google/maps-apis/studio/styles?hl=ko]로 이동하여 원할 경우 프로젝트를 선택합니다.
기존 스타일 중 하나를 선택합니다.
스타일 페이지에서 복제를 선택합니다.
지도 스타일 복제 상자에서 새로운 지도 이름과 원하는 경우 설명을 입력합니다.
저장을 선택합니다.
스타일이 자동으로 게시되고, 새 스타일의 기본 페이지를 볼 수 있습니다. 스타일을 지도 ID와 연결하고 지도에 사용합니다. 자세한 내용은 지도 ID를 지도 스타일에 연결 [https://developers.google.com/maps/documentation/get-map-id?hl=ko#associate-map-id-to-map-style]을 참고하세요.
스타일을 수정하려면 스타일 맞춤설정을 선택하고, 변경한 다음 저장을 선택합니다.
스타일 게시
기존 스타일을 복제하거나 JSON 스타일을 가져와 새 지도 스타일을 처음 만들면 새 지도 스타일의 첫 번째 인스턴스가 자동으로 게시됩니다. 그런 다음 변경사항을 초안으로 저장한 다음 변경사항이 마음에 들 때 게시할 수 있습니다.
게시할 지도 스타일 또는 복제된 지도 스타일을 변경합니다. 자세한 내용은 스타일 변경 [https://developers.google.com/maps/documentation/android-sdk/cloud-customization/map-styles-leg?hl=ko#update-style]을 참고하세요.
변경하는 동안 저장을 선택합니다.
변경사항을 게시하여 해당 지도 스타일을 사용하는 모든 지도 ID에 대해 해당 내용을 적용할 준비가 되면 저장을 선택한 다음 게시를 선택합니다.
게시된 지도 스타일과 초안 지도 스타일 비교
게시됨: 게시된 지도 스타일은 라이브 버전입니다. 이 지도 스타일을 사용하는 모든 지도는 게시된 버전을 표시합니다.
참고: 스타일 변경사항이 앱 및 웹사이트에 반영되는 데는 몇 시간이 걸릴 수 있으며, 앱의 경우 기기가 인터넷에 연결되어 있지 않으면 시간이 더 걸릴 수 있습니다. 몇 시간 후에도 스타일 변경사항이 표시되지 않으면 스타일 변경사항이 업데이트되지 않음 [https://developers.google.com/maps/documentation/android-sdk/cloud-customization/troubleshoot?hl=ko#style-delay]을 참고하세요.
초안: 지도 스타일의 초안 버전은 진행 중인 작업이며 게시할 때까지 지도에 표시되지 않습니다.
스타일 변경
지도 스타일을 변경하려면 스타일 편집기에서 바로 변경할 수 있습니다. 스타일을 업데이트하는 데에는 여러 가지 방법이 있습니다.
준비가 되는 즉시 변경사항을 적용하려는 경우: 스타일 맞춤설정을 선택하고 변경합니다. 저장을 선택하여 변경사항을 초안으로 저장한 다음, 준비가 완료되면 게시를 선택합니다.
먼저 테스트 환경에서 변경 내용을 테스트하려는 경우: 지도 스타일을 바로 변경하는 대신 먼저 스타일을 복제합니다. 자세한 내용은 스타일 복제 [https://developers.google.com/maps/documentation/android-sdk/cloud-customization/map-styles-leg?hl=ko#duplicate-style] 및 지도 스타일 업데이트 테스트 [https://developers.google.com/maps/documentation/android-sdk/cloud-customization/test-style-updates?hl=ko]를 참고하세요.
지도 스타일을 만든 후에는 다음 방법 중 하나로 스타일의 기본 페이지에서 변경할 수 있습니다.
수정: 스타일 맞춤설정을 선택합니다.
복제: 자세한 내용은 스타일 복제 [https://developers.google.com/maps/documentation/android-sdk/cloud-customization/map-styles-leg?hl=ko#duplicate-style]를 참고하세요.
이름 바꾸기: 지도 스타일의 이름을 바꿉니다.
삭제: 지도 스타일을 삭제합니다.
지도 ID 추가: 지도 ID를 스타일과 연결합니다.
주의: 추가하는 지도 ID에 지도 스타일이 아직 할당되어 있지 않은지 또는 지도 ID를 덮어쓰려고 하는지 확인합니다. 지도 스타일 페이지에 지도 ID를 추가하면 해당 지도 ID와 연결된 지도 스타일이 경고 없이 바뀝니다.
스타일에서 지도 ID 연결 또는 삭제
하나의 스타일을 여러 Google 지도 앱에 적용할 수 있으므로 동일한 스타일을 여러 개의 지도 ID와 연결할 수 있습니다. 그러나 지도 ID는 하나의 스타일에만 연결할 수 있습니다.
지도 스타일로 이동하여 스타일을 선택합니다.
지도 ID 추가 또는 지도 ID 수정을 선택합니다.
지도 ID 수정에서 지도 ID 옆의 체크박스를 선택하여 이 스타일과 연결합니다. 지도 ID를 제거하려면 체크박스를 선택 해제하여 연결을 삭제합니다.
주의: 이미 연결된 스타일이 있는 지도 ID를 선택하면 선택 항목에 따라 스타일이 바뀝니다.
저장을 선택합니다.
클라우드 기반 지도 스타일을 사용하려면 앱에 지도 ID를 추가 [https://developers.google.com/maps/documentation/get-map-id?hl=ko#add-a-map-id-to-your-app]합니다.
미리보기 지도에서 탐색
다른 Google 지도에서 탐색하는 것과 동일한 방법으로 미리보기 지도에서 탐색할 수 있습니다.
주소 검색 상자에 주소 또는 위치를 입력합니다.
나침반을 클릭하여 내 위치를 지도 중심에 놓습니다. 위치 정보 액세스 권한을 부여해야 할 수도 있습니다.
지도를 드래그하여 이동합니다.
더하기(+)와 빼기(-)를 사용하여 확대하거나 축소합니다.
지도 지형지물 찾기
스타일을 지정할 지도 지형지물을 찾으려면 지도 지형지물 아래 필터 상자에 검색어를 입력합니다.
지도 지형지물 카테고리를 펼치거나 접어 원하는 지도 지형지물을 찾을 수도 있습니다.
지도 지형지물을 찾을 수 없는 경우:
검색어에 복수형을 사용하지 마세요. 모든 카테고리는 단수형입니다.
지형지물에 더 일반적인 검색어를 사용해 보세요. 예를 들어 '피자 가게'나 '패스트푸드' 대신 '음식점'을 검색하세요.
더 광범위한 카테고리를 사용한 다음 해당 카테고리에서 가장 가까운 지도 지형지물을 검색해 보세요. 예를 들어 '극장' 대신 '엔터테인먼트'를 검색하세요.
지도에서 스타일을 지정할 수 있는 항목 [https://developers.google.com/maps/documentation/android-sdk/cloud-customization/taxonomy?hl=ko]에서 지도 지형지물의 목록을 둘러보세요.
