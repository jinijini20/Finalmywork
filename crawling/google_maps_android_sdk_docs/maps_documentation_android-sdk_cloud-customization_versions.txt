지도 스타일 버전 사용
플랫폼 선택: Android [https://developers.google.com/maps/documentation/android-sdk/cloud-customization/versions?hl=ko] iOS [https://developers.google.com/maps/documentation/ios-sdk/cloud-customization/versions?hl=ko] JavaScript [https://developers.google.com/maps/documentation/javascript/cloud-customization/versions?hl=ko] 웹 서비스 [https://developers.google.com/maps/documentation/maps-static/cloud-customization/versions?hl=ko]
참고: 버전 기록은 2022년 8월 2일 이후에 만든 지도 스타일의 경우에만 확인할 수 있습니다.
지도 스타일을 변경하고 저장하면 변경사항이 저장된 버전이 되어 보거나 복원하거나 복제할 수 있습니다. 스타일의 버전을 보려면 버전 기록을 선택하세요.
지도 스타일의 모든 버전을 보려면 다음 단계를 따르세요.
지도 스타일 [https://console.cloud.google.com/google/maps-apis/studio/styles?hl=ko]로 이동하여 원할 경우 프로젝트를 선택합니다.
기존 지도 스타일을 선택합니다.
스타일 맞춤설정을 선택합니다.
버전 기록을 선택합니다.
버전 기록은 여러 유형으로 나뉩니다.
초안
저장을 선택하면 날짜가 표시된 새 버전이 생성됩니다. 저장된 초안은 명시적으로 게시되기 전에는 앱에서 사용할 수 없습니다.
변경 후 저장하지 않으면 브라우저 창을 닫거나 세션을 종료하는 경우 손실되는 저장되지 않은 변경사항 초안이 생성됩니다.
게시된 버전
새 지도 스타일을 만들거나, 기존 스타일을 복제하거나, JSON 스타일을 가져오면 해당 스타일이 자동으로 게시됩니다.
게시를 선택하면 앱에서 사용할 수 있고 날짜가 표시된 새 버전이 생성됩니다. 가장 최근에 게시된 버전은 연결된 지도 ID가 있는 모든 앱에 적용됩니다.
버전 기록에서 모든 이전 버전을 볼 수 있습니다. 특정 버전을 재사용하고 변경하려면 버전을 복원하여 지도 스타일의 현재, 활성 버전으로 설정하거나, 버전을 복제하여 해당 버전을 기반으로 별도의 새 지도 스타일을 만들어야 합니다.
버전을 복원할 때 저장되지 않은 변경사항은 날짜가 표시되는 새 초안 버전으로 자동 전환됩니다.
중요: 하나 이상의 지도 ID와 연결된 지도 스타일을 게시하면 이 스타일의 게시된 최신 변경사항이 해당 지도 스타일과 연결된 지도 ID를 사용하는 모든 앱에 표시됩니다. 스타일 변경이 완료될 때까지는 스타일 변경사항을 게시하지 말고 저장하세요.
지도 스타일 버전 복제
지도 스타일 [https://console.cloud.google.com/google/maps-apis/studio/styles?hl=ko]로 이동하여 원할 경우 프로젝트를 선택합니다.
기존 스타일 중 하나를 선택하고 스타일 맞춤설정을 클릭합니다.
버전 기록을 선택합니다.
스타일 버전을 선택하고 버전 기록 창 하단에 있는 복제를 선택합니다.
스타일이 복제되고 스타일 이름에 '사본'이 추가됩니다.
복제된 스타일은 자동으로 게시되며 새 탭에서 새 스타일을 열 수 있는 링크가 표시됩니다.
참고: 복제된 스타일에는 복제한 원래 스타일의 버전이 유지되지 않습니다.
지도 스타일 버전 복원
지도 스타일의 이전 버전을 복원하여 사용할 수 있습니다. 예를 들어 특별 이벤트를 지원하기 위해 브랜드 지도 스타일을 만든 경우 이벤트가 끝나면 기본 버전의 지도 스타일을 복원할 수 있습니다.
지도 스타일 [https://console.cloud.google.com/google/maps-apis/studio/styles?hl=ko]로 이동하여 원할 경우 프로젝트를 선택합니다.
원하는 스타일을 선택하고 스타일 맞춤설정을 선택합니다.
버전 기록을 선택합니다.
복원할 버전을 선택하고 버전 기록 창 하단에 있는 복원을 선택합니다. 복원된 스타일이 가장 최신 초안이 됩니다.
버전 기록 창을 닫고, 게시를 선택합니다.
참고: 스타일 변경사항이 앱 및 웹사이트에 반영되는 데는 몇 시간이 걸릴 수 있으며, 앱의 경우 기기가 인터넷에 연결되어 있지 않으면 시간이 더 걸릴 수 있습니다. 몇 시간 후에도 스타일 변경사항이 표시되지 않으면 스타일 변경사항이 업데이트되지 않음 [https://developers.google.com/maps/documentation/android-sdk/cloud-customization/troubleshoot?hl=ko#style-delay]을 참고하세요.
참고: 복원된 버전이 게시된 버전과 정확히 동일한 경우 복원된 버전을 게시할 수 없습니다. 복원된 버전이 게시된 버전과 동일하더라도 복원된 버전의 초안을 게시하려면 어떤 식으로든 스타일을 수정한 다음 저장, 게시를 선택해야 합니다.
