지도 스타일을 최신 버전으로 업데이트
플랫폼 선택: Android [https://developers.google.com/maps/documentation/android-sdk/cloud-customization/update-legacy?hl=ko] iOS [https://developers.google.com/maps/documentation/ios-sdk/cloud-customization/update-legacy?hl=ko] JavaScript [https://developers.google.com/maps/documentation/javascript/cloud-customization/update-legacy?hl=ko] 웹 서비스 [https://developers.google.com/maps/documentation/maps-static/cloud-customization/update-legacy?hl=ko]
Android에서 클라우드 기반 지도 스타일 지정을 사용하려면 Android용 Maps SDK를 위한 최신 렌더기를 사용해야 합니다. 다음 안내를 참고하세요 [https://developers.google.com/maps/documentation/android-sdk/renderer?hl=ko].
잠시 동안 이전 스타일이 계속 작동할 수도 있지만 최대한 빨리 업데이트하는 것이 가장 좋습니다.
지도 스타일을 새 스타일로 업데이트하는 경우 게시하기 전에 스타일 편집기에서 검토하여 스타일을 업데이트한 결과 맞춤 스타일에 원치 않는 영향이 없는지 확인해야 합니다.
업데이트해야 하는 스타일을 식별하려면 지도 스타일에서 '업데이트 가능'을 찾아보세요.
새로운 스타일 지정에서 변경된 사항에 대한 자세한 내용은 지도 지형지물 변경사항 [https://developers.google.com/maps/documentation/android-sdk/cloud-customization/update-old2-leg?hl=ko]을 참고하세요.
주의: 안내에 따라 지도 스타일을 업데이트하고 프로덕션 지도 ID에 할당하기 전에 작동하는지 확인하세요.
새 기본 지도 스타일 미리보기
클라우드 기반 지도 스타일 지정에는 이제 최신 기본 지도 스타일을 포함하여 스타일 업데이트의 미리보기가 포함됩니다. 이 최신 기본 지도에는 Google의 현재 기본 지도와 일치하는 새로운 색상 팔레트 및 지도 환경과 사용성 개선사항이 표시됩니다.
새로고침된 기본 지도를 기존 지도 스타일로 미리 보려면 지도 스타일 업데이트 [https://developers.google.com/maps/documentation/android-sdk/cloud-customization/update-legacy?hl=ko#update-style]에 설명된 대로 지도 스타일을 업데이트하세요.
기존 버전과의 차이점
최신 버전은 다음 기능을 지원하지 않습니다.
업로드된 JSON 스타일에서 새 스타일 만들기. 해결 방법은 JSON 스타일 가져오기 [https://developers.google.com/maps/documentation/android-sdk/cloud-customization/update-legacy?hl=ko#json-import]를 참고하세요.
Google 스타일 템플릿. 이제 새로운 스타일은 모두 기본 지도 스타일에서 시작합니다.
지도 스타일 업데이트
지도 스타일을 업데이트하면 새 지도 스타일이 이전 스타일을 덮어쓰고 게시되지 않은 초안으로 저장합니다. 유일한 예외는 기존 스타일을 업데이트하는 경우로, 이 경우에는 새 스타일이 기존 스타일에 새 기본 지도 스타일을 적용한 사본이 됩니다.
참고: 기존 지도 스타일을 업데이트하면 복제된 지도 스타일은 버전 기록에는 복사되지 않습니다. 적용 가능한 전환된 스타일 지정 구성만 복사됩니다.
업데이트할 수 있는 지도 스타일을 엽니다.
상단의 스타일 업데이트 가능에서 미리보기를 선택하여 기본 지도 미리보기를 엽니다.
기존 지도 스타일이 아닌 지도 스타일의 경우 업데이트를 선택하여 지도 스타일을 업데이트합니다. 업데이트의 결과는 다음과 같습니다.
지도 스타일의 새로운 초안 버전이 생성됩니다.
이전 지도 스타일이 새 지도 스타일로 업데이트됩니다.
기존 지도 스타일의 경우 복제를 선택합니다. 기존 지도 스타일의 복제 결과는 다음과 같습니다.
기존 지도 스타일이 새 지도 스타일로 복제됩니다.
지도 ID는 기존 스타일과 연결된 상태로 유지됩니다.
새 복제된 스타일을 이전 지도 ID 및 데이터 세트와 수동으로 연결해야 합니다.
업데이트가 완료되면 지도 스타일의 업데이트된 초안 버전을 열고 예상대로 작동하는지 확인합니다. 자세한 내용은 업데이트된 지도 스타일 검토 [https://developers.google.com/maps/documentation/android-sdk/cloud-customization/update-legacy?hl=ko#check-style]를 참고하세요.
업데이트된 스타일이 만족스러우면 스타일 편집기에서 게시합니다.
업데이트된 지도 스타일 검토
업데이트된 지도 스타일을 검토하는 방법은 다음과 같습니다.
스타일 편집기에서 지도 스타일의 업데이트된 초안 버전을 엽니다.
맞춤 스타일이 있는 지도 지형지물은 파란색 단색 점으로 표시됩니다. 모든 지도 지형지물을 펼쳐 모든 변경사항을 찾습니다.
다음 영역에서 맞춤 스타일이 있는 모든 지도 지형지물을 확인합니다.
지도 지형지물: 이전 지도 지형지물이 적절한 새 지도 지형지물로 이동했는지 확인합니다.
지도 스타일 지정: 미리보기 지도에서 색상, 공개 상태 및 기타 스타일 지정 기능이 예상대로 렌더링되는지 확인합니다.
새로운 지도 지형지물 활용: 필요에 맞는 더 정확하고 자세한 새로운 지도 지형지물이 있는지 확인합니다.
필요에 따라 업데이트된 지도 스타일을 조정하고 조정된 내용을 테스트합니다.
업데이트된 지도 스타일이 만족스러우면 스타일 편집기에서 게시할 수 있습니다.
JSON 지도 스타일 가져오기
최신 클라우드 스타일 지정에서는 JSON 지도 스타일을 가져올 수 없습니다. 새 클라우드 스타일 지정을 선택 해제했으며 기존 스타일을 아직 이용 중인 일부 사용자의 경우, JSON 스타일을 기존 스타일 편집기로 가져온 다음 업데이트할 수 있습니다.
참고: 이 해결 방법은 기존 지도 스타일이 있으며 선택 해제 옵션이 있는 Google Cloud 프로젝트에만 사용할 수 있습니다. 최신 클라우드 지도 스타일만 있는 Google Cloud 프로젝트에서는 선택 해제를 사용하여 기존 스타일 편집기를 열 수 없습니다.
주의: JSON으로 가져와 새로운 지도 지형지물 계층 구조로 업데이트한 모든 스타일을 테스트하여 예상대로 작동하는지 확인하세요.
스타일 편집기에서 스타일 만들기를 선택한 다음 선택 해제를 선택하여 기존 스타일 편집기를 엽니다.
JSON 스타일 지정 가져오기 [https://developers.google.com/maps/documentation/android-sdk/cloud-customization/map-styles?hl=ko#import-json-styling]의 안내에 따라 기존 지도 스타일을 만듭니다.
가져온 JSON 지도 스타일을 업데이트합니다. 자세한 내용은 스타일 업데이트 [https://developers.google.com/maps/documentation/android-sdk/cloud-customization/update-legacy?hl=ko#update-style]를 참고하세요.
