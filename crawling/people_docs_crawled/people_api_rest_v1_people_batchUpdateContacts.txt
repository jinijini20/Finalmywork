연락처를 일괄적으로 업데이트하고 업데이트된 연락처에 대한 리소스 이름 맵을 PersonResponses에 반환합니다.
지연 시간 및 실패 증가를 방지하려면 동일한 사용자에 대한 변형 요청을 순차적으로 전송해야 합니다.
HTTP 요청
POST https://people.googleapis.com/v1/people:batchUpdateContacts
URL은 gRPC 트랜스코딩 [https://google.aip.dev/127] 구문을 사용합니다.
요청 본문
요청 본문에는 다음과 같은 구조의 데이터가 포함됩니다.
JSON 표현
{
  "contacts": {
    string: {
      object (Person [https://developers.google.com/people/api/rest/v1/people?hl=ko#Person])
    },
    ...
  },
  "updateMask": string,
  "readMask": string,
  "sources": [
    enum (ReadSourceType [https://developers.google.com/people/api/rest/v1/ReadSourceType?hl=ko])
  ]
}
필드
contacts
map (key: string, value: object (Person [https://developers.google.com/people/api/rest/v1/people?hl=ko#Person]))
필수 항목입니다. 업데이트할 개인 데이터에 대한 리소스 이름의 맵입니다. 단일 요청으로 최대 200개의 연락처를 허용합니다.
"key": value 쌍 목록을 포함하는 객체입니다. 예: { "name": "wrench", "mass": "1.3kg", "count": "3" }
