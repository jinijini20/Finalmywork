Geocoding API v4 개요
유럽 경제 지역 (EEA) 개발자
Geocoding API v4의 이번 출시에는 다음 엔드포인트가 포함됩니다.
(미리보기) 주소 지오코딩: 정방향 지오코딩이라고도 합니다.
(미리보기) 위치 지오코딩: 역지오코딩이라고도 합니다.
(미리보기) 장소 지오코딩: 장소 ID에서 주소를 가져옵니다.
(실험 단계) 목적지 검색: 주소, 장소 ID 또는 위치에서 목적지를 검색합니다.
Google Cloud 표준 플랫폼에서 구현
Geocoding API v4는 Google Cloud의 서비스 인프라에 구현됩니다. 이 구현은 OAuth와 같은 향상된 보안 옵션이 있는 더 안전하고 신뢰할 수 있는 플랫폼을 제공합니다. 이 표준 API 설계는 API 전반에 일관성을 제공하여 Geocoding API v4를 사용한 개발의 효율성을 개선합니다.
OAuth 지원 추가
Geocoding API v4는 인증에 OAuth 2.0 사용을 지원합니다. Google은 웹 서버용과 같은 일반적인 OAuth 2.0 시나리오를 지원합니다.
필드 마스크 지원 추가
필드 마스크를 사용하여 응답에서 반환해야 하는 정보를 지정합니다. 주소 지오코딩, 위치 지오코딩, 장소 지오코딩 엔드포인트의 경우 응답 객체의 모든 필드가 기본적으로 반환됩니다. 여행지 검색 엔드포인트의 경우 API 요청에 필드 마스크를 지정해야 합니다.
미리보기 중에는 사용 비용이 청구되지 않습니다.
Geocoding API v4는 GA 전 미리보기 단계에 있으며 미리보기 기간에는 API 사용에 대한 요금이 청구되지 않습니다. API 호출은 Cloud 콘솔의 결제 페이지에 표시되지만 요금은 0달러입니다.
