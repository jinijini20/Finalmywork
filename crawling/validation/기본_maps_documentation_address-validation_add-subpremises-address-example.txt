주소에 부속 건물 추가 - 예시 (미국만 해당)
bookmark_border
이 페이지의 내용
일반적인 예: 하위 건물 추가
특이 사례 예: 하위 건물 추가
누락된 하위 구역과 추론 및 교체된 구성요소
이 문서에서는 주소 유효성 검사 API가 시스템에서 하위 건물 추가 동작을 보장하는 응답 신호를 제공하는 여러 실제 시나리오를 설명합니다. 이러한 신호는 미국 주소에만 사용할 수 있습니다. 컨텍스트는 유효성 검사 로직 빌드의 워크플로 예시를 참고하세요.
참고: 여기에 나온 예는 설명용이며 모든 시나리오를 다루지는 않습니다.
일반적인 예: 하위 건물 추가
이 시나리오에서는 시스템에서 고객에게 주소에 호수를 추가하라는 메시지를 표시할 수 있는 주소를 보여줍니다.
주소 입력됨 지역
1450 Brickell Avenue, Miami, FL 33131-4065 미국
하위 건물 누락 주소에 대한 평결
아래 예에서는 중요한 신호를 강조 표시합니다.
{
  "inputGranularity": "PREMISE",
  "validationGranularity": "PREMISE",
  "geocodeGranularity": "PREMISE",
  "possibleNextAction": "CONFIRM_ADD_SUBPREMISES"
}
작업: 이 주소의 경우 사용자에게 호수 번호를 추가하라는 메시지를 표시할 수 있습니다.
특이 사례 예: 하위 건물 추가
다음 예에서는 추가 조사가 필요한 주소 품질 문제를 verdict가 나타내는 상황을 다룹니다. 이 예에서는 시스템 로직을 개선하기 위해 판결에서 주소 구성요소로 로직을 이동하여 더 완전한 그림을 얻는 방법도 보여줍니다.
누락된 하위 구역과 추론 및 교체된 구성요소
이 예시에서는 지역이 누락되고 우편번호가 잘못된 미국 주소를 입력하는 방법을 보여줍니다.
주소 입력됨 지역
1450 Brickell Avenue, FL 33132-4065 미국
누락된 하위 구역과 추론되고 대체된 구성요소에 대한 평결
{
  "inputGranularity": "PREMISE",
  "validationGranularity": "PREMISE",
  "geocodeGranularity": "PREMISE",
  "hasInferredComponents": true,
  "hasReplacedComponents": true,
  "possibleNextAction": "CONFIRM_ADD_SUBPREMISES"
}
주소 구성요소를 추가로 조사한 결과, 지역이 추론되었고 우편번호가 대체된 것으로 확인되었습니다.
{
   "componentName": {
     "text": "33131",
   }
   "componentType": "postal_code",
   "confirmationLevel": "CONFIRMED",
   "replaced": true
},
{
   "componentName": {
     "text": "Miami",
     "languageCode": "en"
   }
   "componentType": "locality",
   "confirmationLevel": "CONFIRMED",
   "inferred": true
}
작업: 이 주소의 경우 사용자에게 호수 번호를 추가하라는 메시지를 표시하거나 전체 주소를 검토하라는 메시지를 표시할 수 있습니다.
