data[]
object (GridData [https://developers.google.com/workspace/sheets/api/reference/rest/v4/spreadsheets/sheets?hl=ko#GridData])
그리드 시트인 경우 그리드의 데이터
반환되는 GridData 객체 수는 이 시트에서 요청된 범위 수에 따라 다릅니다. 예를 들어 이 값이 Sheet1를 나타내고 스프레드시트가 Sheet1!A1:C10 및 Sheet1!D15:E20 범위로 요청된 경우 첫 번째 GridData에는 0의 startRow [https://developers.google.com/workspace/sheets/api/reference/rest/v4/spreadsheets/sheets?hl=ko#GridData.FIELDS.start_row]/startColumn [https://developers.google.com/workspace/sheets/api/reference/rest/v4/spreadsheets/sheets?hl=ko#GridData.FIELDS.start_column]가 있고 두 번째 GridData에는 startRow 14(0부터 시작하는 행 15) 및 startColumn 3(0부터 시작하는 열 D)가 있습니다.
DATA_SOURCE [https://developers.google.com/workspace/sheets/api/reference/rest/v4/spreadsheets/sheets?hl=ko#SheetType] 시트의 경우 특정 범위를 요청할 수 없으며 GridData에 모든 값이 포함됩니다.

필드
sheetId
integer
시트의 ID입니다. Must be non-negative. 이 필드는 설정된 후에는 변경할 수 없습니다.
title
string
시트의 이름입니다.
index
integer
스프레드시트 내 시트의 색인입니다. 시트 속성을 추가하거나 업데이트할 때 이 필드를 제외하면 시트가 추가되거나 시트 목록의 끝으로 이동합니다. 시트 색인을 업데이트하거나 시트를 삽입할 때 이동은 '이전의 이동' 색인에서 고려됩니다. 예를 들어 시트가 3개 (S1, S2, S3) 있는 경우 S1을 S2 앞에 이동하려면 색인을 2로 설정해야 합니다. 요청된 색인이 시트의 현재 색인과 동일하거나 요청된 새 색인이 현재 시트 색인 + 1과 같은 경우 시트 색인 업데이트 요청은 무시됩니다.
sheetType
enum (SheetType [https://developers.google.com/workspace/sheets/api/reference/rest/v4/spreadsheets/sheets?hl=ko#SheetType])
시트 유형입니다. 기본값은 GRID [https://developers.google.com/workspace/sheets/api/reference/rest/v4/spreadsheets/sheets?hl=ko#SheetType]입니다. 이 필드는 설정된 후에는 변경할 수 없습니다.
gridProperties
object (GridProperties [https://developers.google.com/workspace/sheets/api/reference/rest/v4/spreadsheets/sheets?hl=ko#GridProperties])
이 시트가 그리드인 경우 시트의 추가 속성입니다. 시트가 차트나 이미지가 포함된 객체 시트인 경우 이 필드는 표시되지 않습니다. 작성 시 그리드가 아닌 시트에 그리드 속성을 설정하면 오류가 발생합니다.
이 시트가 DATA_SOURCE [https://developers.google.com/workspace/sheets/api/reference/rest/v4/spreadsheets/sheets?hl=ko#SheetType] 시트인 경우 이 필드는 출력 전용이지만 데이터 소스 시트가 UI에서 렌더링되는 방식을 반영하는 속성(예: rowCount [https://developers.google.com/workspace/sheets/api/reference/rest/v4/spreadsheets/sheets?hl=ko#GridProperties.FIELDS.row_count])을 포함합니다.

boolean

UI의 탭 색상입니다. 지원 중단됨: tabColorStyle [https://developers.google.com/workspace/sheets/api/reference/rest/v4/spreadsheets/sheets?hl=ko#SheetProperties.FIELDS.tab_color_style]를 사용하세요.

PERCENTILE 보간 지점은 조건부 서식 범위의 모든 셀에 대해 지정된 백분위수입니다. 이는 값이 =PERCENTILE(FLATTEN(range), value / 100)인 경우 NUMBER와 같습니다. 여기서 평탄화 시 범위의 오류는 무시됩니다.

필드
headerColorStyle
object (ColorStyle [https://developers.google.com/workspace/sheets/api/reference/rest/v4/spreadsheets/other?hl=ko#ColorStyle])
헤더 행의 색상입니다. 이 필드를 설정하면 헤더 행이 지정된 색상으로 채워집니다. 그렇지 않으면 헤더 행이 기본 색상으로 채워집니다.
firstBandColorStyle
object (ColorStyle [https://developers.google.com/workspace/sheets/api/reference/rest/v4/spreadsheets/other?hl=ko#ColorStyle])
교체되는 첫 번째 색상입니다. 이 필드를 설정하면 첫 번째 밴딩된 행이 지정된 색상으로 채워집니다. 그렇지 않으면 첫 번째 밴딩된 행이 기본 색상으로 채워집니다.
secondBandColorStyle
object (ColorStyle [https://developers.google.com/workspace/sheets/api/reference/rest/v4/spreadsheets/other?hl=ko#ColorStyle])
교차되는 두 번째 색상입니다. 이 필드를 설정하면 두 번째 밴딩된 행이 지정된 색상으로 채워집니다. 그렇지 않으면 두 번째 밴딩된 행이 기본 색상으로 채워집니다.
footerColorStyle
object (ColorStyle [https://developers.google.com/workspace/sheets/api/reference/rest/v4/spreadsheets/other?hl=ko#ColorStyle])
마지막 행의 색상입니다. 이 필드를 설정하지 않으면 바닥글이 추가되지 않고 마지막 행은 이전 행의 색상에 따라 firstBandColorStyle [https://developers.google.com/workspace/sheets/api/reference/rest/v4/spreadsheets/sheets?hl=ko#TableRowsProperties.FIELDS.first_band_color_style] 또는 secondBandColorStyle [https://developers.google.com/workspace/sheets/api/reference/rest/v4/spreadsheets/sheets?hl=ko#TableRowsProperties.FIELDS.second_band_color_style]로 채워집니다. 바닥글이 없는 기존 테이블을 바닥글이 있는 테이블로 업데이트하면 범위가 1행 확장됩니다. 기존 테이블을 업데이트하여 바닥글을 추가하고 바닥글을 삭제하면 범위가 1행 줄어듭니다.
