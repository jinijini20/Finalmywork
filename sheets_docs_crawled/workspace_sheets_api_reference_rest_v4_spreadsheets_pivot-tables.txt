map (key: integer, value: object ( PivotFilterCriteria [https://developers.google.com/workspace/sheets/api/reference/rest/v4/spreadsheets/pivot-tables?hl=ko#PivotFilterCriteria]))
이 항목은 지원 중단되었습니다.
소스 열 오프셋별 필터 매핑(선택사항).
필터는 데이터를 피벗 테이블로 집계하기 전에 적용됩니다. 맵의 키는 필터링하려는 소스 범위의 열 오프셋이고 값은 해당 열의 기준입니다.
예를 들어 소스가 C10:E15이면 0 키에는 C 열의 필터가 있고 1 키에는 D 열의 필터가 있습니다.
이 필드는 지원 중단되었으며 filterSpecs [https://developers.google.com/workspace/sheets/api/reference/rest/v4/spreadsheets/pivot-tables?hl=ko#PivotTable.FIELDS.filter_specs]로 대체되었습니다.

|        | Feb |
|        | Mar |
+--------+-----+
| Q1 Total     |
+--------------+

ManualRule
소스 데이터 열의 값을 원하는 이름을 가진 버킷으로 수동으로 정리할 수 있습니다. 예를 들어 주별로 인구를 집계하는 피벗 테이블은 다음과 같습니다.
+-------+-------------------+
| State | SUM of Population |
+-------+-------------------+
| AK    |               0.7 |
| AL    |               4.8 |
| AR    |               2.9 |
...
+-------+-------------------+
수동 그룹 규칙에 그룹 목록 (예: groupName = 'Central', items = ['AL', 'AR', 'IA', ...])을 제공하여 시간대별 인구를 집계하는 피벗 테이블로 변환할 수 있습니다. 소스 데이터에 시간대 열을 추가하고 피벗 테이블을 조정하여 유사한 효과를 얻을 수도 있습니다.
+-----------+-------------------+
| Time Zone | SUM of Population |
+-----------+-------------------+
| Central   |             106.3 |
| Eastern   |             151.9 |
| Mountain  |              17.4 |
...
+-----------+-------------------+

HistogramRule
소스 데이터 열의 숫자 값을 일정한 크기의 버킷으로 구성할 수 있습니다. HistogramRule.start [https://developers.google.com/workspace/sheets/api/reference/rest/v4/spreadsheets/pivot-tables?hl=ko#HistogramRule.FIELDS.start]부터 HistogramRule.end [https://developers.google.com/workspace/sheets/api/reference/rest/v4/spreadsheets/pivot-tables?hl=ko#HistogramRule.FIELDS.end]까지의 모든 값은 크기가 HistogramRule.interval [https://developers.google.com/workspace/sheets/api/reference/rest/v4/spreadsheets/pivot-tables?hl=ko#HistogramRule.FIELDS.interval]인 그룹에 배치됩니다. 또한 HistogramRule.start [https://developers.google.com/workspace/sheets/api/reference/rest/v4/spreadsheets/pivot-tables?hl=ko#HistogramRule.FIELDS.start] 미만의 모든 값은 한 그룹에 배치되고 HistogramRule.end [https://developers.google.com/workspace/sheets/api/reference/rest/v4/spreadsheets/pivot-tables?hl=ko#HistogramRule.FIELDS.end] 초과 값은 다른 그룹에 배치됩니다. HistogramRule.interval [https://developers.google.com/workspace/sheets/api/reference/rest/v4/spreadsheets/pivot-tables?hl=ko#HistogramRule.FIELDS.interval]만 필요하지만 HistogramRule.start [https://developers.google.com/workspace/sheets/api/reference/rest/v4/spreadsheets/pivot-tables?hl=ko#HistogramRule.FIELDS.start] 및 HistogramRule.end [https://developers.google.com/workspace/sheets/api/reference/rest/v4/spreadsheets/pivot-tables?hl=ko#HistogramRule.FIELDS.end]가 모두 제공된 경우 HistogramRule.start [https://developers.google.com/workspace/sheets/api/reference/rest/v4/spreadsheets/pivot-tables?hl=ko#HistogramRule.FIELDS.start]이 HistogramRule.end [https://developers.google.com/workspace/sheets/api/reference/rest/v4/spreadsheets/pivot-tables?hl=ko#HistogramRule.FIELDS.end]보다 작아야 합니다. 예를 들어 행이 50개가 넘는 연령별 평균 구매 금액을 보여주는 피벗 테이블은 다음과 같습니다.
+-----+-------------------+
| Age | AVERAGE of Amount |
+-----+-------------------+
| 16  |            $27.13 |
| 17  |             $5.24 |
| 18  |            $20.15 |
...
+-----+-------------------+
HistogramRule.start [https://developers.google.com/workspace/sheets/api/reference/rest/v4/spreadsheets/pivot-tables?hl=ko#HistogramRule.FIELDS.start]이 25이고 HistogramRule.interval [https://developers.google.com/workspace/sheets/api/reference/rest/v4/spreadsheets/pivot-tables?hl=ko#HistogramRule.FIELDS.interval]이 20이며 HistogramRule.end [https://developers.google.com/workspace/sheets/api/reference/rest/v4/spreadsheets/pivot-tables?hl=ko#HistogramRule.FIELDS.end]이 65인 히스토그램 그룹 규칙을 적용하여 아래와 같은 피벗 테이블로 변환할 수 있습니다.
+-------------+-------------------+
| Grouped Age | AVERAGE of Amount |
+-------------+-------------------+
| < 25        |            $19.34 |
| 25-45       |            $31.43 |
| 45-65       |            $35.87 |
| > 65        |            $27.55 |
+-------------+-------------------+
| Grand Total |            $29.12 |
+-------------+-------------------+
JSON 표현
{
  "interval": number,
  "start": number,
  "end": number
}
필드
interval
number
생성된 버킷의 크기입니다. 양수여야 합니다.
start
number
항목이 일정한 크기의 버킷에 배치되는 최솟값입니다. 시작 아래의 값은 하나의 버킷으로 그룹화됩니다. 이 필드는 선택사항입니다.
end
number
항목이 일정한 크기의 버킷에 배치되는 최대 값입니다. 끝 값보다 큰 값은 하나의 버킷으로 그룹화됩니다. 이 필드는 선택사항입니다.

YEAR_MONTH DateTimeRuleType [https://developers.google.com/workspace/sheets/api/reference/rest/v4/spreadsheets/pivot-tables?hl=ko#DateTimeRuleType]로 날짜-시간 그룹 규칙을 적용하면 다음과 같은 피벗 테이블이 생성됩니다.
+--------------+--------------+
| Grouped Date | SUM of Sales |
+--------------+--------------+

적용할 날짜 시간 그룹화 유형입니다.
DateTimeRuleType
사용 가능한 날짜 시간 그룹화 규칙 유형입니다. 이 문서에서는 스프레드시트 언어가 'en-US'라고 가정하지만, 날짜 및 시간의 실제 렌더링은 일부 규칙 유형에 스프레드시트의 언어를 사용합니다.
열거형
DATE_TIME_RULE_TYPE_UNSPECIFIED 기본 유형이며 사용하지 마세요.
SECOND 0~59초 단위로 날짜를 그룹화합니다.
MINUTE 0~59분으로 날짜를 그룹화합니다.
HOUR 0~23 사이의 24시간제를 사용하여 시간별로 날짜를 그룹화합니다.
HOUR_MINUTE 24시간 시스템(예: 19:45)을 사용하여 시간과 분으로 날짜를 그룹화합니다.
HOUR_MINUTE_AMPM 12시간제를 사용하여 시간과 분으로 날짜를 그룹화합니다(예:오후 7시 45분). AM/PM 지정은 스프레드시트 언어를 기반으로 번역됩니다.
DAY_OF_WEEK 일요일과 같이 요일별로 날짜를 그룹화합니다. 요일은 스프레드시트 언어를 기반으로 번역됩니다.
DAY_OF_YEAR 1~366 사이의 연중 일로 날짜를 그룹화합니다. 2월 29일 이후의 날짜는 윤년과 윤년이 아닌 해에 따라 다른 버킷에 속합니다.
DAY_OF_MONTH 1~31일의 일별로 날짜를 그룹화합니다.
DAY_MONTH 날짜를 일과 월별로 그룹화합니다(예: 11월 22일). 월은 스프레드시트 언어에 따라 번역됩니다.
MONTH 날짜를 월별로 그룹화합니다(예: 11월). 월은 스프레드시트 언어에 따라 번역됩니다.
QUARTER 분기별로 날짜를 그룹화합니다(예: 1분기(1월~3월)).
YEAR 연도별로 날짜를 그룹화합니다(예: 2008년).
YEAR_MONTH 연도 및 월별로 날짜를 그룹화합니다(예: 2008-Nov). 월은 스프레드시트 언어에 따라 번역됩니다.
YEAR_QUARTER 연도 및 분기별로 날짜를 그룹화합니다(예: 2008년 4분기).
YEAR_MONTH_DAY 연도, 월, 일(예: 2008-11-22)별로 날짜를 그룹화합니다.
